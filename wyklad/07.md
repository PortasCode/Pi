## 1. Czym jest strumień i struktura FILE

### Teoria

Strumień (ang. *stream*) to abstrakcja przepływu danych — sekwencja bajtów płynąca z jakiegoś źródła (klawiatura, plik, sieć) do jakiegoś celu (ekran, plik, pamięć). Dzięki temu programista nie musi wiedzieć, czy czyta z dysku, czy z klawiatury — interfejs jest zawsze taki sam.

W języku C strumień reprezentowany jest przez wskaźnik na strukturę `FILE`. Struktura ta jest wewnętrzna — nie zaglądamy do jej pól ręcznie — ale zawiera:

- **deskryptor pliku** — liczba całkowita, którą system operacyjny przydziela każdemu otwartemu plikowi (0 = stdin, 1 = stdout, 2 = stderr).
- **bufor danych** — obszar pamięci, do którego trafiają bajty zanim zostaną faktycznie zapisane na dysk lub wysłane na ekran; buforowanie przyspiesza I/O bo redukuje liczbę wywołań systemowych.
- **wskaźnik pozycji** — informacja, w którym miejscu pliku aktualnie się znajdujemy (przesunięcie od początku w bajtach).
- **tryb otwarcia** — np. `"r"`, `"wb"`, `"a+"`.
- **flagi błędów i EOF** — bity mówiące, czy podczas operacji wystąpił błąd albo osiągnięto koniec pliku.

Trzy strumienie są otwierane automatycznie przy starcie każdego programu C:

| Nazwa | Znaczenie | Domyślne podłączenie |
|-------|-----------|----------------------|
| `stdin` | standardowe wejście | klawiatura |
| `stdout` | standardowe wyjście | ekran (terminal) |
| `stderr` | standardowe wyjście błędów | ekran (terminal) |

Standard C gwarantuje, że jednocześnie można mieć otwartych co najmniej `FOPEN_MAX` plików (co najmniej 8, w praktyce na Linuksie zwykle 1024).

### Kod z komentarzami

```c
#include <stdio.h>   /* nagłówek ze wszystkimi funkcjami I/O */

int main(void) {
    /* stdin, stdout, stderr to globalne wskaźniki FILE* dostępne zawsze */
    printf("Piszemy na stdout\n");          /* stdout — domyślne wyjście */
    fprintf(stderr, "Błąd na stderr\n");    /* stderr — wyjście błędów */

    /* Sprawdzamy makro FOPEN_MAX — minimalny limit otwartych plików */
    printf("FOPEN_MAX = %d\n", FOPEN_MAX);

    /* Sprawdzamy rozmiar bufora standardowego */
    printf("BUFSIZ = %d\n", BUFSIZ);

    return 0;
}
```

### Co program wypisze

```
Piszemy na stdout
Błąd na stderr
FOPEN_MAX = 16
BUFSIZ = 8192
```

*(Wartości `FOPEN_MAX` i `BUFSIZ` zależą od platformy i kompilatora.)*

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| Używanie `FILE f;` zamiast `FILE *f` | Błąd kompilacji lub UB | `FILE` jest nieprzezroczyste — zawsze wskaźnik |
| Nieodczytanie wartości zwracanej przez funkcje I/O | Ciche błędy | Zawsze sprawdzaj wynik `fopen`, `fread`, `fwrite` |
| Pisanie na `stderr` przez `printf` zamiast `fprintf(stderr,...)` | Błędy pojawiają się na stdout, mieszają z normalnym wyjściem | Używaj `fprintf(stderr, ...)` do diagnostyki |

## 2. Otwieranie i zamykanie pliku — fopen / fclose

### Teoria

Zanim możemy cokolwiek czytać z pliku lub do niego pisać, musimy go **otworzyć** — powiedzieć systemowi operacyjnemu, że chcemy z tym plikiem pracować. System zwraca nam wtedy "uchwyt" (`FILE*`), którego będziemy używać do wszystkich dalszych operacji.

**`fopen`** — otwiera plik i zwraca wskaźnik `FILE*`. Jeśli plik nie może zostać otwarty (nie istnieje, brak uprawnień), zwraca `NULL`. Zawsze trzeba to sprawdzić.

**`fclose`** — zamyka strumień: opróżnia bufor (zapisuje niewypisane dane na dysk) i zwalnia zasoby. Niezamknięcie pliku to wyciek zasobów — system operacyjny ma ograniczony limit otwartych plików.

Sygnatury:
```c
FILE *fopen(const char *restrict nazwa, const char *restrict tryb);
int   fclose(FILE *strumień);
```

`fclose` zwraca `0` przy sukcesie, `EOF` przy błędzie.

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>  /* exit(), EXIT_FAILURE */

int main(void) {
    FILE *fp;  /* wskaźnik na strumień — na razie nieokreślony */

    /* Otwieramy plik "dane.txt" do zapisu ("w" — write) */
    /* Jeśli plik istnieje — jego zawartość zostanie skasowana */
    /* Jeśli nie istnieje — zostanie utworzony */
    fp = fopen("dane.txt", "w");

    /* ZAWSZE sprawdzamy, czy fopen się udało */
    if (fp == NULL) {
        /* perror drukuje systemowy opis błędu */
        perror("Blad otwierania pliku do zapisu");
        exit(EXIT_FAILURE);  /* kończymy program z kodem błędu */
    }

    /* Zapisujemy kilka linii do pliku */
    fprintf(fp, "Linia pierwsza\n");   /* jak printf, ale do pliku */
    fprintf(fp, "Linia druga\n");
    fprintf(fp, "Liczba: %d\n", 42);

    /* Zamykamy plik — OBOWIĄZKOWE */
    /* fclose opróżnia bufor, więc dane faktycznie trafiają na dysk */
    if (fclose(fp) != 0) {
        perror("Blad zamykania pliku");
        exit(EXIT_FAILURE);
    }

    /* Teraz otwieramy ten sam plik do odczytu ("r" — read) */
    fp = fopen("dane.txt", "r");

    if (fp == NULL) {
        perror("Blad otwierania pliku do odczytu");
        exit(EXIT_FAILURE);
    }

    char linia[100];  /* bufor na jedną linię — 100 znaków wystarczy */

    /* Czytamy linie aż do końca pliku */
    /* fgets zwraca NULL gdy koniec pliku lub błąd */
    while (fgets(linia, sizeof(linia), fp) != NULL) {
        printf("Przeczytano: %s", linia);  /* linia już zawiera '\n' */
    }

    fclose(fp);  /* zawsze zamykamy */

    return 0;
}
```

### Co program wypisze

```
Przeczytano: Linia pierwsza
Przeczytano: Linia druga
Przeczytano: Liczba: 42
```

*(Na dysku powstaje plik `dane.txt` z trzema liniami.)*

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| Brak sprawdzenia `fp == NULL` | Crash (segfault) przy próbie użycia | Zawsze `if (fp == NULL)` po `fopen` |
| Niezamknięcie pliku | Utrata danych (bufor nie opróżniony), wyciek zasobów | Używaj `fclose` zawsze, nawet przy błędzie — rozważ `goto cleanup` |
| Użycie `fp` po `fclose` | Niezdefiniowane zachowanie (UB) | Po `fclose` ustaw `fp = NULL` |
| Zapomnienie o `\n` w `fprintf` | Dane mogą zostać wypisane bez nowej linii | Świadome pisanie formatów |

## 3. Tryby dostępu do pliku

### Teoria

Drugi argument `fopen` to **tryb** — łańcuch znaków opisujący, co chcemy robić z plikiem. Dobór trybu jest ważny: zły tryb może skasować dane albo uniemożliwić odczyt.

| Tryb | Nazwa | Co robi | Plik musi istnieć? |
|------|-------|---------|-------------------|
| `"r"` | read | Odczyt od początku | TAK |
| `"w"` | write | Zapis — kasuje istniejącą zawartość | NIE (tworzy) |
| `"a"` | append | Dopisuje na końcu | NIE (tworzy) |
| `"r+"` | read/write | Odczyt i zapis, nie kasuje | TAK |
| `"w+"` | write/read | Odczyt i zapis, kasuje zawartość | NIE (tworzy) |
| `"a+"` | append/read | Odczyt i dopisywanie | NIE (tworzy) |

Modyfikatory dodawane do trybu:

- **`b`** — tryb binarny (np. `"rb"`, `"wb"`). W trybie tekstowym na Windows `\n` jest konwertowane na `\r\n` i z powrotem. Tryb binarny wyłącza tę konwersję. Na Linuksie nie ma różnicy, ale pisanie `b` jest dobrą praktyką dla przenośności.
- **`x`** — tryb wyłączności (od C11): `"wx"` tworzy plik tylko jeśli nie istnieje — jeśli istnieje, zwraca błąd. Zapobiega przypadkowemu nadpisaniu.
- **`+`** — tryb odczytu/zapisu.

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    FILE *fp;

    /* --- Tryb "w" — tworzy nowy plik lub kasuje istniejący --- */
    fp = fopen("test_trybow.txt", "w");
    if (!fp) { perror("w"); exit(1); }
    fputs("Pierwsza linia\n", fp);  /* zapisujemy tekst */
    fputs("Druga linia\n",    fp);
    fclose(fp);
    printf("Tryb 'w': plik zapisany\n");

    /* --- Tryb "a" — dopisuje na końcu, nie kasuje --- */
    fp = fopen("test_trybow.txt", "a");
    if (!fp) { perror("a"); exit(1); }
    fputs("Trzecia linia (dopisana)\n", fp);
    fclose(fp);
    printf("Tryb 'a': linia dopisana\n");

    /* --- Tryb "r" — tylko odczyt --- */
    fp = fopen("test_trybow.txt", "r");
    if (!fp) { perror("r"); exit(1); }
    char buf[128];
    printf("--- Zawartosc pliku ---\n");
    while (fgets(buf, sizeof(buf), fp))
        printf("%s", buf);
    fclose(fp);

    /* --- Tryb "wx" — wyłączność: błąd jeśli plik istnieje --- */
    fp = fopen("test_trybow.txt", "wx");
    if (fp == NULL) {
        /* Oczekiwany błąd — plik już istnieje */
        printf("Tryb 'wx': plik juz istnieje, fopen zwrocil NULL (poprawnie)\n");
    } else {
        fclose(fp);  /* nie powinniśmy tu trafić */
    }

    /* --- Tryb "rb" — binarny odczyt (ważne przy plikach binarnych) --- */
    fp = fopen("test_trybow.txt", "rb");
    if (!fp) { perror("rb"); exit(1); }
    printf("Tryb 'rb': plik otwarty binarnie\n");
    fclose(fp);

    return 0;
}
```

### Co program wypisze

```
Tryb 'w': plik zapisany
Tryb 'a': linia dopisana
--- Zawartosc pliku ---
Pierwsza linia
Druga linia
Trzecia linia (dopisana)
Tryb 'wx': plik juz istnieje, fopen zwrocil NULL (poprawnie)
Tryb 'rb': plik otwarty binarnie
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| `"w"` zamiast `"a"` gdy chcemy dopisać | Kasowanie całej zawartości pliku | Dokładnie przemyśl tryb; `"a"` = append |
| `"r"` na nieistniejący plik | `fopen` zwraca `NULL` | Sprawdzaj wynik `fopen` |
| Brak `b` przy plikach binarnych na Windows | Uszkodzenie danych przez konwersję `\r\n` | Zawsze `"rb"` / `"wb"` dla plików binarnych |
| `"wx"` nieobsługiwane na starszych kompilatorach | Błąd kompilacji | Wymaga C11 lub nowszego |

## 4. Wykrywanie końca pliku — feof i EOF

### Teoria

Podczas czytania pliku w pętli musimy wiedzieć, kiedy skończyły się dane. W C mamy dwa mechanizmy:

**Makro `EOF`** — specjalna wartość całkowita (zwykle `-1`), którą funkcje odczytujące znaki (`fgetc`, `getc`, `getchar`) zwracają gdy osiągną koniec pliku lub wystąpi błąd. Dlatego zmienna przechowująca wynik musi być `int`, a nie `char` — `char` może nie pomieścić wartości `EOF`.

**Funkcja `feof(FILE *strumien)`** — zwraca niezerową wartość (prawda), jeśli wewnętrzna flaga EOF strumienia jest ustawiona. Flaga ta ustawia się dopiero **po** nieudanej próbie odczytu — nie "z wyprzedzeniem". Dlatego idiom `while(!feof(fp))` jest często błędny — może spowodować przetworzenie ostatnich danych dwa razy lub wejście w nieskończoną pętlę przy błędzie I/O.

**Zalecany idiom**: sprawdzaj wynik samego `fgetc` / `fgets` / `fread`:
```c
while ((c = fgetc(fp)) != EOF) { ... }
while (fgets(buf, sizeof(buf), fp) != NULL) { ... }
```

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    /* Tworzymy plik testowy */
    FILE *fp = fopen("eof_test.txt", "w");
    if (!fp) { perror("zapis"); exit(1); }
    fputs("Ala\nma\nkota\n", fp);
    fclose(fp);

    /* --- Poprawny idiom z fgetc --- */
    fp = fopen("eof_test.txt", "r");
    if (!fp) { perror("odczyt"); exit(1); }

    int c;          /* MUSI być int, nie char — EOF to -1 */
    int licznik = 0;

    /* Pętla kończy się gdy fgetc zwróci EOF (koniec pliku lub błąd) */
    while ((c = fgetc(fp)) != EOF) {
        putchar(c);   /* wypisujemy każdy bajt na stdout */
        licznik++;
    }

    /* Teraz sprawdzamy DLACZEGO wyszliśmy z pętli */
    if (feof(fp)) {
        printf("\n[Osiagnieto koniec pliku po %d znakach]\n", licznik);
    }
    if (ferror(fp)) {
        printf("\n[Wystapil blad I/O]\n");
    }

    fclose(fp);

    /* --- Błędny idiom — NIE RÓB TAK --- */
    /* Poniższy kod może przetworzyć ostatnią linię dwa razy */
    /*
    fp = fopen("eof_test.txt", "r");
    char buf[128];
    while (!feof(fp)) {          // ZLE: feof jest true dopiero PO nieudanym odczycie
        fgets(buf, sizeof(buf), fp);
        printf("%s", buf);
    }
    fclose(fp);
    */

    /* --- Poprawny idiom z fgets --- */
    fp = fopen("eof_test.txt", "r");
    if (!fp) { perror("fgets"); exit(1); }

    char buf[128];
    printf("--- Odczyt przez fgets ---\n");

    /* fgets zwraca NULL przy EOF lub błędzie — to jest właściwy warunek stopu */
    while (fgets(buf, sizeof(buf), fp) != NULL) {
        printf("Linia: [%s]", buf);  /* buf już zawiera '\n' */
    }

    fclose(fp);
    return 0;
}
```

### Co program wypisze

```
Ala
ma
kota

[Osiagnieto koniec pliku po 12 znakach]
--- Odczyt przez fgets ---
Linia: [Ala
]
Linia: [ma
]
Linia: [kota
]
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| `char c = fgetc(fp)` zamiast `int c` | Na platformach z unsigned char nigdy nie zobaczymy EOF | Zawsze `int c` |
| `while(!feof(fp))` jako jedyny warunek | Podwójne przetworzenie ostatniej porcji danych | Sprawdzaj wynik samej funkcji odczytującej |
| Ignorowanie `ferror` | Nie wiemy czy wyszliśmy z pętli przez EOF czy błąd | Po pętli sprawdzaj zarówno `feof` jak i `ferror` |

## 5. Buforowanie strumienia — fflush, setbuf, setvbuf

### Teoria

Buforowanie polega na tym, że dane zapisane przez program nie trafiają od razu na dysk (lub ekran), lecz gromadzą się w obszarze pamięci zwanym **buforem**. Dopiero gdy bufor się zapełni, zawiera znak nowej linii (buforowanie liniowe) lub program jawnie opróżni bufor, dane faktycznie "wychodzą" do systemu operacyjnego.

**Po co buforowanie?** Odczyt/zapis bajtów jeden po jednym przez system operacyjny byłby bardzo wolny — każde wywołanie systemowe kosztuje czas. Buforowanie grupuje wiele małych operacji w jedną dużą, co jest znacznie efektywniejsze.

Trzy tryby buforowania:
- **`_IOFBF`** (fully buffered) — dane wychodzą tylko gdy bufor jest pełny lub wywołamy `fflush`. Domyślnie dla plików dyskowych.
- **`_IOLBF`** (line buffered) — dane wychodzą po każdej linii (`\n`). Domyślnie dla `stdout` gdy jest podłączony do terminala.
- **`_IONBF`** (no buffering) — każdy bajt natychmiast. Domyślnie dla `stderr`.

**`fflush(fp)`** — natychmiast opróżnia bufor wyjściowy strumienia `fp`. Jeśli przekazać `NULL`, opróżnia wszystkie otwarte strumienie wyjściowe. Standardowo `fflush(stdin)` jest niezdefiniowane (nie rób tego — zamiast tego używaj `while(getchar() != '\n');` by wyczyścić bufor wejściowy).

**`setbuf(fp, buf)`** — ustawia bufor ręcznie: `buf` musi mieć rozmiar `BUFSIZ` lub przekazać `NULL` (wyłącza buforowanie).

**`setvbuf(fp, buf, tryb, rozmiar)`** — pełna kontrola: typ bufora + rozmiar. Musi być wywołane **przed** jakąkolwiek operacją I/O na strumieniu.

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(void) {
    /* --- Demonstracja fflush --- */
    /* Bez \n tekst może nie pojawić się natychmiast na ekranie */
    printf("Ładuję dane");
    fflush(stdout);   /* wymuś wypisanie nawet bez \n */
    /* Tutaj mogłoby być długie obliczenie */
    printf(" — gotowe!\n");

    /* --- Tworzenie pliku z własnym buforowaniem --- */
    FILE *fp = fopen("bufor_test.txt", "w");
    if (!fp) { perror("fopen"); exit(1); }

    /* Własny bufor o rozmiarze 256 bajtów, tryb pełny */
    char moj_bufor[256];

    /* setvbuf MUSI być wywołane przed pierwszą operacją I/O */
    if (setvbuf(fp, moj_bufor, _IOFBF, sizeof(moj_bufor)) != 0) {
        fprintf(stderr, "setvbuf nie powiodl sie\n");
    }

    fprintf(fp, "Linia 1\n");
    fprintf(fp, "Linia 2\n");
    /* Dane siedzą w buforze, na dysku ich jeszcze nie ma */

    fflush(fp);   /* teraz trafiają na dysk */
    fprintf(fp, "Linia 3 (po fflush)\n");

    fclose(fp);   /* fclose automatycznie robi fflush przed zamknięciem */

    /* --- Wyłączanie buforowania (tryb _IONBF) --- */
    FILE *fp2 = fopen("niebuf_test.txt", "w");
    if (!fp2) { perror("fopen2"); exit(1); }

    /* NULL jako bufor + _IONBF = brak buforowania */
    setvbuf(fp2, NULL, _IONBF, 0);

    /* Każdy zapis trafia od razu na dysk */
    fprintf(fp2, "Bez buforowania\n");
    fclose(fp2);

    /* --- Czyszczenie bufora wejściowego po scanf --- */
    int wiek;
    printf("Podaj wiek (wpisz liczbe i Enter): ");
    /* Normalnie byśmy czytali od użytkownika, tutaj symulujemy */
    /* Po scanf('\n' zostaje w buforze stdin) */
    /* Prawidłowe czyszczenie bufora stdin: */
    /* while (getchar() != '\n');  */
    /* (nie używamy fflush(stdin) — to UB w standardzie C) */

    printf("Czyszczenie bufora stdin: while(getchar()!='\\n') jest prawidlowe\n");
    printf("fflush(stdin) jest niezdefiniowane w standardzie C!\n");

    return 0;
}
```

### Co program wypisze

```
Ładuję dane — gotowe!
Czyszczenie bufora stdin: while(getchar()!='\n') jest prawidlowe
fflush(stdin) jest niezdefiniowane w standardzie C!
```

*(Na dysku powstają pliki `bufor_test.txt` i `niebuf_test.txt`.)*

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| `fflush(stdin)` | Niezdefiniowane zachowanie (UB) — na Windows "działa", na Linuksie nie | Używaj `while(getchar() != '\n');` |
| `setvbuf` po pierwszej operacji I/O | Niezdefiniowane zachowanie | Wywołuj `setvbuf` jako pierwsze po `fopen` |

## 6. Odczyt i zapis pojedynczego znaku — fgetc / fputc / getchar / putchar

### Teoria

Najbardziej elementarne operacje na strumieniu to odczyt lub zapis **jednego znaku** (jednego bajtu).

- **`fgetc(fp)`** — funkcja czyta jeden znak ze strumienia `fp`, zwraca jego kod jako `int`. Przy błędzie lub EOF zwraca `EOF` (makro, zwykle `-1`).
- **`getc(fp)`** — to samo co `fgetc`, ale zaimplementowane jako **makro** (może wielokrotnie ewaluować argument — nie przekazuj wyrażeń z efektami ubocznymi jak `getc(fp++)`).
- **`getchar()`** — odpowiednik `getc(stdin)`.
- **`fputc(c, fp)`** — funkcja zapisuje znak o kodzie `c` do strumienia `fp`, zwraca `c` lub `EOF` przy błędzie.
- **`putc(c, fp)`** — makro, odpowiednik `fputc`.
- **`putchar(c)`** — odpowiednik `putc(c, stdout)`.
- **`ungetc(c, fp)`** — "odkłada" znak z powrotem do strumienia, tak jakby nie był jeszcze odczytany. Gwarantowany jest jeden poziom "odłożenia". Przydatne w parserach.

Pamiętaj: zmienna na wynik `fgetc` musi być `int`, nie `char`!

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    /* --- Zapis znaków do pliku przez fputc --- */
    FILE *fp = fopen("znaki.txt", "w");
    if (!fp) { perror("zapis"); exit(1); }

    /* Zapisujemy znaki A, B, C, D, E i nową linię */
    for (int i = 0; i < 5; i++) {
        int wynik = fputc('A' + i, fp);  /* 'A'=65, 'B'=66, ... */
        if (wynik == EOF) {
            fprintf(stderr, "Blad zapisu znaku!\n");
        }
    }
    fputc('\n', fp);  /* nowa linia na końcu */
    fclose(fp);

    /* --- Odczyt znaków z pliku przez fgetc --- */
    fp = fopen("znaki.txt", "r");
    if (!fp) { perror("odczyt"); exit(1); }

    int c;    /* MUSI być int — EOF to -1, nie mieści się w char */

    printf("Znaki z pliku: ");
    while ((c = fgetc(fp)) != EOF) {
        if (c == '\n') {
            printf("[ENTER]");  /* pokazujemy znak nowej linii explicite */
        } else {
            putchar(c);         /* wypisuje znak na stdout */
        }
    }
    printf("\n");

    fclose(fp);

    /* --- Demonstracja ungetc --- */
    fp = fopen("znaki.txt", "r");
    if (!fp) { perror("ungetc"); exit(1); }

    c = fgetc(fp);  /* czytamy 'A' */
    printf("Przeczytany znak: %c (kod %d)\n", c, c);

    ungetc(c, fp);  /* "odkładamy" 'A' z powrotem do bufora strumienia */

    c = fgetc(fp);  /* czytamy ponownie — dostajemy znowu 'A' */
    printf("Po ungetc ponownie: %c (kod %d)\n", c, c);

    fclose(fp);

    /* --- getchar: czyta ze stdin --- */
    /* (zakomentowane żeby program nie czekał na wejście) */
    /* printf("Wpisz znak: "); */
    /* int wczytany = getchar(); */
    /* printf("Wczytano: %c\n", wczytany); */

    return 0;
}
```

### Co program wypisze

```
Znaki z pliku: ABCDE[ENTER]
Przeczytany znak: A (kod 65)
Po ungetc ponownie: A (kod 65)
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| `char c = fgetc(fp)` | Na platformach unsigned char, EOF (255 lub -1) może być równy jakiemuś znakowi | `int c = fgetc(fp)` |
| `getc(fp++)` lub podobne wyrażenia z efektem ubocznym | `getc` to makro — argument może być obliczony wielokrotnie | Używaj `fgetc` gdy argument ma efekt uboczny |
| Wiele `ungetc` pod rząd | Standard gwarantuje tylko jeden poziom | Używaj `ungetc` raz między odczytami |

## 7. Odczyt i zapis łańcucha znaków — fgets / fputs / puts

### Teoria

Zamiast czytać/pisać po jednym znaku, częściej operujemy na całych **liniach tekstu**.

- **`fgets(str, n, fp)`** — czyta co najwyżej `n-1` znaków ze strumienia `fp` do tablicy `str`, zatrzymuje się na `\n` (i go wczytuje!) lub na EOF. Zawsze dopisuje terminator `\0`. Zwraca `str` lub `NULL` przy EOF/błędzie. To bezpieczna wersja przestarzałego `gets` (które nie sprawdzało rozmiaru bufora i jest usuniete w C11).

- **`fputs(str, fp)`** — zapisuje łańcuch `str` do strumienia `fp`. W przeciwieństwie do `puts` NIE dopisuje automatycznie `\n`. Zwraca `EOF` przy błędzie lub nieujemną wartość przy sukcesie.

- **`puts(str)`** — jak `fputs(str, stdout)`, ale **automatycznie dopisuje `\n`** na końcu.

Uwaga na `fgets`: jeśli linia jest krótsza niż `n-1`, w buforze znajdzie się `\n`. Jeśli jest dłuższa, reszta pozostanie w strumieniu i będzie czekać na następne wywołanie `fgets`. Znak `\n` można usunąć ręcznie.

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>  /* strlen */

int main(void) {
    /* --- Zapis przez fputs i puts --- */
    FILE *fp = fopen("lancuchy.txt", "w");
    if (!fp) { perror("zapis"); exit(1); }

    /* fputs NIE dodaje \n — musimy sami */
    fputs("Pierwsza linia\n", fp);
    fputs("Druga linia\n",    fp);
    fputs("Trzecia linia\n",  fp);

    fclose(fp);

    /* puts na stdout DODAJE \n automatycznie */
    puts("To jest puts — automatycznie dodaje nową linię");

    /* --- Odczyt przez fgets --- */
    fp = fopen("lancuchy.txt", "r");
    if (!fp) { perror("odczyt"); exit(1); }

    char bufor[50];   /* bufor na maksymalnie 49 znaków + '\0' */
    int numer = 1;

    while (fgets(bufor, sizeof(bufor), fp) != NULL) {
        /* fgets zostawia '\n' w buforze — pozbywamy się go */
        size_t dlugosc = strlen(bufor);
        if (dlugosc > 0 && bufor[dlugosc - 1] == '\n') {
            bufor[dlugosc - 1] = '\0';  /* zastąp '\n' terminatorem */
        }

        printf("Linia %d: [%s] (dlugosc %zu)\n",
               numer, bufor, strlen(bufor));
        numer++;
    }

    fclose(fp);

    /* --- Demonstracja: co gdy linia jest za długa dla bufora --- */
    fp = fopen("lancuchy.txt", "w");
    if (!fp) { perror("dlugi"); exit(1); }
    fputs("Krótka\nBardzo długaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa linia\nKoniec\n", fp);
    fclose(fp);

    fp = fopen("lancuchy.txt", "r");
    if (!fp) { perror("dlugi2"); exit(1); }

    char maly_bufor[20];   /* tylko 19 znaków */

    printf("\n--- Czytanie małym buforem (20 znaków) ---\n");
    while (fgets(maly_bufor, sizeof(maly_bufor), fp) != NULL) {
        /* Sprawdzamy czy linia była dłuższa niż bufor */
        int pelna_linia = (strchr(maly_bufor, '\n') != NULL);
        printf("Fragment [%s]%s\n",
               maly_bufor,
               pelna_linia ? " (pełna linia)" : " (linia obcięta!)");
    }

    fclose(fp);
    return 0;
}
```

### Co program wypisze

```
To jest puts — automatycznie dodaje nową linię
Linia 1: [Pierwsza linia] (dlugosc 14)
Linia 2: [Druga linia] (dlugosc 11)
Linia 3: [Trzecia linia] (dlugosc 13)

--- Czytanie małym buforem (20 znaków) ---
Fragment [Krótka
] (pełna linia)
Fragment [Bardzo długaaaaaa] (linia obcięta!)
Fragment [aaaaaaaaaaaaaaa l] (linia obcięta!)
Fragment [inia
] (pełna linia)
Fragment [Koniec
] (pełna linia)
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| `gets(buf)` zamiast `fgets` | Przepełnienie bufora — podatność bezpieczeństwa, usunięte w C11 | Zawsze `fgets(buf, sizeof(buf), stdin)` |
| Zakładanie braku `\n` na końcu bufora `fgets` | Niespodziewane zachowanie przy porównaniach | Zawsze usuń `\n`: `buf[strcspn(buf,"\n")] = 0;` |
| `fputs` zamiast `puts` gdy chcemy `\n` | Brak nowej linii w wyjściu | Pamiętaj: `puts` dodaje `\n`, `fputs` nie |
| Za mały bufor dla `fgets` | Linia dzielona na kawałki w kolejnych wywołaniach | Sprawdzaj obecność `\n` w buforze |

## 8. Pozycja w pliku — fseek / ftell / rewind / fgetpos

### Teoria

Każdy otwarty plik ma **wskaźnik pozycji** — liczbę mówiącą, na którym bajcie aktualnie się znajdujemy. Kolejne operacje odczytu/zapisu automatycznie go przesuwają. Możemy jednak skoczyć do dowolnej pozycji.

- **`fseek(fp, offset, skad)`** — przesuwa wskaźnik pozycji o `offset` bajtów względem punktu `skad`:
  - `SEEK_SET` — od początku pliku
  - `SEEK_CUR` — od bieżącej pozycji
  - `SEEK_END` — od końca pliku
  - Dla plików tekstowych `offset` powinien być `0` (aby przejść do punktu) lub wartością uzyskaną z `ftell`. Dowolne przesunięcia bezpieczne są tylko dla plików binarnych.

- **`ftell(fp)`** — zwraca bieżącą pozycję jako `long int` (liczba bajtów od początku pliku binarnego). Przy błędzie zwraca `-1L`.

- **`rewind(fp)`** — ustawia pozycję na początku pliku, odpowiednik `fseek(fp, 0L, SEEK_SET)` z wyczyszczeniem flag błędów.

- **`fgetpos(fp, &pos)` / `fsetpos(fp, &pos)`** — bezpieczna alternatywa dla plików o rozmiarze powyżej 2 GB (używają `fpos_t` zamiast `long`). Zwracają `0` przy sukcesie, niezerowe przy błędzie.

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    /* Tworzymy binarny plik z 5 liczbami int */
    FILE *fp = fopen("pozycja.bin", "wb");
    if (!fp) { perror("zapis binarny"); exit(1); }

    int dane[5] = {10, 20, 30, 40, 50};
    fwrite(dane, sizeof(int), 5, fp);  /* zapisujemy 5 elementów */
    fclose(fp);

    /* Otwieramy do odczytu binarnego */
    fp = fopen("pozycja.bin", "rb");
    if (!fp) { perror("odczyt binarny"); exit(1); }

    /* Sprawdzamy aktualną pozycję (powinna być 0 — początek) */
    long pos = ftell(fp);
    printf("Pozycja po otwarciu: %ld bajtow\n", pos);

    int wartosc;

    /* Skaczemy do 3. elementu (indeks 2, od 0) */
    /* offset = 2 * sizeof(int) bajtów od początku */
    fseek(fp, 2 * (long)sizeof(int), SEEK_SET);
    pos = ftell(fp);
    printf("Po fseek na element [2]: pozycja %ld bajtow\n", pos);

    fread(&wartosc, sizeof(int), 1, fp);  /* czytamy element [2] */
    printf("Element [2] = %d (oczekiwany: 30)\n", wartosc);

    /* Teraz jesteśmy za elementem [2], pos = 3 * sizeof(int) */
    pos = ftell(fp);
    printf("Pozycja po odczycie: %ld\n", pos);

    /* Cofamy się o 1 element od bieżącej pozycji */
    fseek(fp, -(long)sizeof(int), SEEK_CUR);
    fread(&wartosc, sizeof(int), 1, fp);
    printf("Element po cofnieciu o 1: %d (oczekiwany: 30)\n", wartosc);

    /* Skaczemy do ostatniego elementu (sizeof(int) przed końcem) */
    fseek(fp, -(long)sizeof(int), SEEK_END);
    fread(&wartosc, sizeof(int), 1, fp);
    printf("Ostatni element: %d (oczekiwany: 50)\n", wartosc);

    /* rewind — wracamy na sam początek */
    rewind(fp);
    pos = ftell(fp);
    printf("Po rewind: pozycja %ld\n", pos);

    /* Sprawdzamy rozmiar pliku: skocz na koniec, odczytaj pozycję */
    fseek(fp, 0L, SEEK_END);
    long rozmiar = ftell(fp);
    printf("Rozmiar pliku: %ld bajtow (%ld liczb int po %zu bajtow)\n",
           rozmiar, rozmiar / (long)sizeof(int), sizeof(int));

    fclose(fp);
    return 0;
}
```

### Co program wypisze

*(zakładając `sizeof(int) == 4`)*

```
Pozycja po otwarciu: 0 bajtow
Po fseek na element [2]: pozycja 8 bajtow
Element [2] = 30 (oczekiwany: 30)
Pozycja po odczycie: 12
Element po cofnieciu o 1: 30 (oczekiwany: 30)
Ostatni element: 50 (oczekiwany: 50)
Po rewind: pozycja 0
Rozmiar pliku: 20 bajtow (5 liczb int po 4 bajtow)
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| `fseek` z niezerowym offset na pliku tekstowym | Niezdefiniowane zachowanie | W plikach tekstowych `fseek(fp, 0, SEEK_SET/END)` albo wartość z `ftell` |
| `ftell` na pliku > 2 GB zwraca błąd | `long` może nie pomieścić pozycji | Użyj `fgetpos/fsetpos` lub `ftello` (POSIX) |
| Pomijanie sprawdzenia wyniku `fseek` | Ciche błędy przy pliku tekstowym lub zamkniętym strumieniu | Sprawdzaj: `if (fseek(...) != 0) ...` |

## 9. Odczyt i zapis bloków binarnych — fread / fwrite

### Teoria

`fread` i `fwrite` służą do odczytu/zapisu **bloków surowych danych** (bajtów) — bez żadnej interpretacji, konwersji ani buforowania na poziomie tekstu. Są niezbędne do pracy z plikami binarnymi, np. własnymi formatami danych, strukturami, tablicami liczb.

```c
size_t fwrite(const void *wsk, size_t rozmiar, size_t ilosc, FILE *fp);
size_t fread (      void *wsk, size_t rozmiar, size_t ilosc, FILE *fp);
```

Argumenty:
- `wsk` — wskaźnik do danych (w pamięci)
- `rozmiar` — rozmiar jednego elementu w bajtach (zwykle `sizeof(typ)`)
- `ilosc` — ile elementów zapisać/odczytać
- `fp` — strumień

Funkcje zwracają liczbę **faktycznie** przetworzonych elementów. Jeśli jest mniejsza niż `ilosc`, nastąpił błąd lub koniec pliku.

**Ważne**: Pliki binarne mogą nie być przenośne między platformami o różnej kolejności bajtów (endianness) lub różnych rozmiarach typów. Jeśli potrzebujesz przenośności, zapisuj dane w formacie tekstowym lub używaj stałych rozmiarów (np. `int32_t` z `<stdint.h>`).

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

/* Definicja struktury — taki "rekord" będziemy zapisywać */
typedef struct {
    int    id;         /* identyfikator, 4 bajty */
    float  wynik;      /* wynik punktowy, 4 bajty */
    char   imie[20];   /* imię, 20 bajtów */
} Student;

int main(void) {
    /* --- Zapis tablicy struktur do pliku binarnego --- */
    Student studenci[3] = {
        {1, 85.5f, "Ala"},
        {2, 92.0f, "Bolek"},
        {3, 78.3f, "Celina"}
    };

    FILE *fp = fopen("studenci.bin", "wb");  /* "wb" — write binary */
    if (!fp) { perror("zapis binarny"); exit(1); }

    /* Zapisujemy całą tablicę jednym wywołaniem */
    /* rozmiar = sizeof(Student), ilosc = 3 */
    size_t zapisano = fwrite(studenci, sizeof(Student), 3, fp);
    printf("Zapisano %zu rekordow (oczekiwano 3)\n", zapisano);

    if (zapisano != 3) {
        fprintf(stderr, "Blad zapisu!\n");
    }

    fclose(fp);

    /* --- Odczyt całej tablicy --- */
    Student wczytani[3];  /* bufor na 3 struktury */

    fp = fopen("studenci.bin", "rb");  /* "rb" — read binary */
    if (!fp) { perror("odczyt binarny"); exit(1); }

    size_t wczytano = fread(wczytani, sizeof(Student), 3, fp);
    printf("Wczytano %zu rekordow\n", wczytano);

    for (size_t i = 0; i < wczytano; i++) {
        printf("Student %d: id=%d, wynik=%.1f, imie=%s\n",
               (int)i + 1,
               wczytani[i].id,
               wczytani[i].wynik,
               wczytani[i].imie);
    }

    fclose(fp);

    /* --- Odczyt i zapis po jednym rekordzie (sekwencyjnie) --- */
    fp = fopen("studenci.bin", "rb");
    if (!fp) { perror("sekwencja"); exit(1); }

    Student jeden;
    int numer = 1;

    printf("\n--- Odczyt po jednym rekordzie ---\n");
    /* fread zwraca 0 gdy koniec pliku lub błąd */
    while (fread(&jeden, sizeof(Student), 1, fp) == 1) {
        printf("Rekord %d: [%s] — %.1f pkt\n", numer++, jeden.imie, jeden.wynik);
    }

    fclose(fp);

    /* --- Sprawdzanie rozmiaru pliku przez fseek+ftell --- */
    fp = fopen("studenci.bin", "rb");
    if (!fp) { perror("rozmiar"); exit(1); }

    fseek(fp, 0L, SEEK_END);          /* skocz na koniec */
    long rozmiar = ftell(fp);          /* odczytaj pozycję = rozmiar */
    long ile_rekordow = rozmiar / (long)sizeof(Student);

    printf("\nRozmiar pliku: %ld bajtow\n", rozmiar);
    printf("Rozmiar struktury: %zu bajtow\n", sizeof(Student));
    printf("Liczba rekordow: %ld\n", ile_rekordow);

    fclose(fp);
    return 0;
}
```

### Co program wypisze

```
Zapisano 3 rekordow (oczekiwano 3)
Wczytano 3 rekordow
Student 1: id=1, wynik=85.5, imie=Ala
Student 2: id=2, wynik=92.0, imie=Bolek
Student 3: id=3, wynik=78.3, imie=Celina

--- Odczyt po jednym rekordzie ---
Rekord 1: [Ala] — 85.5 pkt
Rekord 2: [Bolek] — 92.0 pkt
Rekord 3: [Celina] — 78.3 pkt

Rozmiar pliku: 84 bajtow
Rozmiar struktury: 28 bajtow
Liczba rekordow: 3
```

*(Rozmiar struktury może się różnić przez padding — `sizeof(Student)` może wynosić 28 lub 32 w zależności od kompilatora.)*

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| `fread/fwrite` w trybie tekstowym | Konwersja `\r\n` może uszkodzić dane binarne | Zawsze `"rb"` / `"wb"` dla danych binarnych |
| Niesprawdzanie wartości zwrotnej | Ciche błędy częściowego zapisu/odczytu | Sprawdzaj czy wynik == `ilosc` |
| Struktory z wskaźnikami (`char *name`) | Wskaźnik jest liczbą — po wczytaniu wskazuje na nieistniejący adres | Używaj tablic znaków (`char name[N]`), nie wskaźników |
| Zakładanie stałego sizeof między platformami | Plik binarny nieprzenośny | Używaj `int32_t`, `float` (IEEE 754 jest standardem) |

## 10. Formatowany odczyt i zapis — fprintf / fscanf / printf / scanf

### Teoria

`printf` i `scanf` to najczęściej używane funkcje I/O w C. Ich wersje z przedrostkiem `f` działają tak samo, ale na dowolnym strumieniu (pierwszym argumentem jest `FILE*`).

**Składnia formatu `printf`/`fprintf`:**
```
%[flagi][szerokość_pola][.precyzja][rozmiar]specyfikator
```

Flagi:
- `+` — zawsze wypisuj znak (+ lub -)
- `-` — wyrównaj do lewej
- `0` — uzupełnij zerami (zamiast spacjami)
- `#` — alternatywna forma (np. `0x` dla hex)
- spacja — wstaw spację przed liczbą bez znaku

Modyfikatory rozmiaru:
- `hh` — `signed char` / `unsigned char`
- `h` — `short`
- `l` — `long` / `double` (przy `%f`)
- `ll` — `long long`
- `z` — `size_t`
- `t` — `ptrdiff_t`

Specyfikatory:
- `d`, `i` — liczba całkowita ze znakiem
- `u` — unsigned int
- `o`, `x`, `X` — octal, hex małe, hex duże
- `f`, `F` — float/double dziesiętny
- `e`, `E` — notacja naukowa
- `g`, `G` — krótszy z `%f` i `%e`
- `c` — znak
- `s` — łańcuch
- `p` — wskaźnik
- `%` — dosłowny znak `%`

**`scanf`/`fscanf`** — odczyt sformatowany. Zwraca liczbę faktycznie przypisanych pól. Ważne: zmienne przekazujemy przez wskaźnik (`&zmienna`).

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    /* --- Demonstracja formatowania printf --- */
    int    liczba_c = 255;
    double pi       = 3.14159265358979;
    char   imie[]   = "Bolek";

    /* Podstawowe formaty */
    printf("Dziesiętny:    %d\n",  liczba_c);     /* 255 */
    printf("Ósemkowy:      %o\n",  liczba_c);     /* 377 */
    printf("Szesnastkowy:  %x\n",  liczba_c);     /* ff */
    printf("Hex z 0x:      %#x\n", liczba_c);     /* 0xff */
    printf("Szesnastkowy:  %X\n",  liczba_c);     /* FF */

    /* Formatowanie liczb zmiennoprzecinkowych */
    printf("Domyślnie:     %f\n",   pi);           /* 3.141593 */
    printf("2 miejsca:     %.2f\n", pi);           /* 3.14 */
    printf("Naukowy:       %e\n",   pi);           /* 3.141593e+00 */
    printf("Krótszy:       %g\n",   pi);           /* 3.14159 */

    /* Szerokość pola i wyrównanie */
    printf("Prawo [%10d]\n",  liczba_c);           /* [       255] */
    printf("Lewo  [%-10d]\n", liczba_c);           /* [255       ] */
    printf("Zera  [%010d]\n", liczba_c);           /* [0000000255] */
    printf("Plus  [%+d]\n",   liczba_c);           /* [+255] */

    /* Łańcuchy */
    printf("Imię: [%s]\n",     imie);              /* [Bolek] */
    printf("Szer: [%10s]\n",   imie);              /* [     Bolek] */
    printf("Obci: [%.3s]\n",   imie);              /* [Bol] */

    /* size_t — użyj %zu */
    size_t rozmiar = sizeof(double);
    printf("sizeof(double) = %zu bajtow\n", rozmiar);  /* 8 */

    /* --- fprintf do pliku --- */
    FILE *fp = fopen("wyniki.txt", "w");
    if (!fp) { perror("wyniki"); exit(1); }

    fprintf(fp, "%-10s %5s %8s\n", "Imię", "Wiek", "Wynik");
    fprintf(fp, "%-10s %5d %8.2f\n", "Ala",   22, 85.5);
    fprintf(fp, "%-10s %5d %8.2f\n", "Bolek", 21, 92.0);
    fprintf(fp, "%-10s %5d %8.2f\n", "Celina",23, 78.3);

    fclose(fp);
    printf("\nPlik wyniki.txt zapisany\n");

    /* --- fscanf z pliku --- */
    fp = fopen("wyniki.txt", "r");
    if (!fp) { perror("czytanie"); exit(1); }

    char   n[11];
    int    wiek;
    double wynik;
    char   naglowek[50];

    /* Pomijamy nagłówek */
    fgets(naglowek, sizeof(naglowek), fp);
    printf("\n--- Dane z pliku (fscanf) ---\n");

    /* fscanf zwraca liczbę wczytanych pól */
    /* %10s — łańcuch bez spacji, max 10 znaków */
    while (fscanf(fp, "%10s %d %lf", n, &wiek, &wynik) == 3) {
        printf("  %s: wiek=%d, wynik=%.1f\n", n, wiek, wynik);
    }

    fclose(fp);
    return 0;
}
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| `scanf("%f", &x)` gdzie `x` to `double` | Niezdefiniowane zachowanie — `%f` to `float*` | Dla `double` użyj `%lf` w `scanf` (w `printf` `%f` obsługuje oba) |

| Ignorowanie wartości zwracanej `scanf` | Czytanie niezainicjalizowanych danych | `if (scanf(...) != n_pol) { obsługa błędu }` |

## 11. Operacje na łańcuchach znaków — biblioteka string.h

### Teoria

Biblioteka `<string.h>` zawiera funkcje do pracy z tablicami `char` zakończonymi `'\0'`. Działają **tylko na typie `char`** — dla Unicode istnieją inne biblioteki.

**Kopiowanie:**
- `strcpy(cel, zrodlo)` — kopiuje, nie sprawdza rozmiaru — niebezpieczna przy zbyt małym buforze!
- `strncpy(cel, zrodlo, n)` — kopiuje co najwyżej `n` znaków; uwaga: może nie dodać `'\0'` jeśli `zrodlo` ma >= `n` znaków
- `strdup(s)` — alokuje dynamicznie nową kopię łańcucha (pamiętaj o `free`!)
- `strndup(s, n)` — jak `strdup` ale co najwyżej `n` znaków

**Łączenie:**
- `strcat(cel, zrodlo)` — dołącza `zrodlo` na końcu `cel`; niebezpieczna
- `strncat(cel, zrodlo, n)` — bezpieczniejsza wersja; zawsze dodaje `'\0'`

**Porównanie:**
- `strcmp(s1, s2)` — zwraca 0 jeśli równe, <0 jeśli s1 < s2, >0 jeśli s1 > s2
- `strncmp(s1, s2, n)` — porównuje co najwyżej `n` znaków
- `strcoll(s1, s2)` — jak `strcmp` ale uwzględnia locale (alfabety narodowe)

**Wyszukiwanie:**
- `strchr(s, c)` — wskaźnik na pierwsze wystąpienie znaku `c` w `s` lub NULL
- `strrchr(s, c)` — wskaźnik na ostatnie wystąpienie
- `strstr(s1, s2)` — wskaźnik na pierwsze wystąpienie podłańcucha `s2` w `s1` lub NULL
- `strpbrk(s1, s2)` — wskaźnik na pierwsze wystąpienie dowolnego znaku z `s2` w `s1`

**Długość i inne:**
- `strlen(s)` — długość łańcucha (bez `'\0'`)
- `strcspn(s1, s2)` — długość prefixu `s1` złożonego ze znaków **nie** będących w `s2`
- `strspn(s1, s2)` — długość prefixu `s1` złożonego **tylko** ze znaków z `s2`

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(void) {
    char s1[50] = "Ala ma kota";
    char s2[50];
    const char *p;

    /* --- strlen --- */
    printf("strlen(\"%s\") = %zu\n", s1, strlen(s1));  /* 11 */

    /* --- strcpy --- */
    strcpy(s2, s1);   /* kopiuje s1 do s2 */
    printf("Po strcpy s2 = \"%s\"\n", s2);

    /* --- strcat --- */
    strcat(s2, " i psa");   /* dołącza do s2 */
    printf("Po strcat s2 = \"%s\"\n", s2);  /* "Ala ma kota i psa" */

    /* --- strcmp --- */
    int cmp = strcmp("jabłko", "gruszka");
    printf("strcmp(\"jabłko\",\"gruszka\") = %d (%s)\n",
           cmp, cmp > 0 ? "jabłko > gruszka" : "jabłko <= gruszka");

    printf("strcmp(\"abc\",\"abc\") = %d\n", strcmp("abc", "abc"));  /* 0 */

    /* --- strchr --- */
    p = strchr(s1, 'm');   /* szuka 'm' */
    if (p) {
        printf("Pierwsze 'm' w \"%s\" na pozycji %td\n",
               s1, p - s1);  /* ptrdiff_t: odległość między wskaźnikami */
    }

    /* --- strrchr --- */
    p = strrchr(s1, 'a');   /* szuka ostatniego 'a' */
    if (p) {
        printf("Ostatnie 'a' na pozycji %td\n", p - s1);
    }

    /* --- strstr --- */
    p = strstr(s1, "ma");   /* szuka podłańcucha "ma" */
    if (p) {
        printf("Podłańcuch \"ma\" znaleziony na pozycji %td\n", p - s1);
    }

    /* --- strncpy z ręcznym '\0' --- */
    char bufor[5];
    strncpy(bufor, "ABCDEFGH", 4);  /* kopiuje tylko 4 znaki */
    bufor[4] = '\0';                /* strncpy może nie dodać '\0' ! */
    printf("strncpy 4 znaki: \"%s\"\n", bufor);   /* "ABCD" */

    /* --- strdup — dynamiczna kopia --- */
    char *kopia = strdup("Dynamiczna kopia");
    if (kopia == NULL) { perror("strdup"); exit(1); }
    printf("strdup: \"%s\"\n", kopia);
    free(kopia);   /* MUSIMY zwolnić pamięć alokowaną przez strdup */
    kopia = NULL;  /* zabezpieczenie przed double-free */

    /* --- strcspn i strspn --- */
    const char *tekst   = "Ala123 ma kota";
    const char *cyfry   = "0123456789";
    const char *litery  = "AaBbCcDdEeFfGgHhIiJjKkLlMmNnOoP"
                          "pQqRrSsTtUuVvWwXxYyZz";

    size_t dlugosc_liter  = strspn(tekst, litery);   /* ile liter na początku */
    size_t do_cyfry       = strcspn(tekst, cyfry);   /* ile znaków przed pierwszą cyfrą */

    printf("strspn (tylko litery): %zu\n",  dlugosc_liter);  /* 3 (Ala) */
    printf("strcspn (do cyfry):    %zu\n",  do_cyfry);       /* 3 (Ala) */

    return 0;
}
```

```

Po strcpy s2 = "Ala ma kota"
Po strcat s2 = "Ala ma kota i psa"
strcmp("jabłko","gruszka") = 5 (jabłko > gruszka)

Ostatnie 'a' na pozycji 10
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| `strcpy(cel, zrodlo)` gdy cel jest za mały | Przepełnienie bufora — klasyczna podatność | Sprawdzaj `strlen(zrodlo) < sizeof(cel)` lub używaj `strncpy` + ręczny `'\0'` |
| `strncpy` bez ręcznego `'\0'` | Brak terminatora, UB w dalszych operacjach | `buf[n-1] = '\0';` zawsze po `strncpy` |
| `strdup` bez `free` | Wyciek pamięci | Każde `strdup` musi mieć odpowiadające `free` |
| `strcmp(s1, s2) == true` | Niespodziewane zachowanie — `true` to 1, a `strcmp` zwraca 0 dla równości | Zawsze `strcmp(s1,s2) == 0` dla równości |
| Używanie `strlen` na niezainicjalizowanej tablicy | UB — szukanie `'\0'` w losowej pamięci | Inicjalizuj tablice lub sprawdzaj przed użyciem |

## 12. Rozpoznawanie i klasyfikacja znaków — ctype.h i wctype.h

### Teoria

`<ctype.h>` zawiera funkcje do sprawdzania kategorii znaku `char` i do konwersji wielkości liter. Wszystkie działają na **kodach ASCII** (lub bieżącym locale dla `char`), **nie** na Unicode.

Dla Unicode (znaki `wchar_t`) istnieje `<wctype.h>` z analogicznymi funkcjami z prefiksem `isw` / `tow`.

Ważne: argument musi być `unsigned char` lub `EOF` — przekazanie `char` ze znakiem ujemnym (polskie litery w Latin-2) to undefined behaviour! Rzutuj: `isalpha((unsigned char)c)`.

| Funkcja z ctype.h | Zwraca prawdę gdy |
|-------------------|------------------|
| `isalpha(c)` | litera (a-z, A-Z) |
| `isdigit(c)` | cyfra (0-9) |
| `isalnum(c)` | litera lub cyfra |
| `isspace(c)` | biały znak (spacja, `\t`, `\n`, `\r`, `\f`, `\v`) |
| `isupper(c)` | duża litera |
| `islower(c)` | mała litera |
| `ispunct(c)` | znak interpunkcyjny |
| `isprint(c)` | znak drukowalny (łącznie ze spacją) |
| `iscntrl(c)` | znak kontrolny |
| `isxdigit(c)` | cyfra szesnastkowa (0-9, a-f, A-F) |
| `tolower(c)` | zamienia na małą literę |
| `toupper(c)` | zamienia na dużą literę |

### Kod z komentarzami

```c
#include <stdio.h>
#include <ctype.h>   /* isalpha, isdigit, toupper, ... */
#include <string.h>  /* strlen */

int main(void) {
    const char *tekst = "Hello, World! 123";

    int liter = 0, cyfr = 0, spacji = 0, innych = 0;

    printf("Analiza: \"%s\"\n", tekst);

    for (size_t i = 0; i < strlen(tekst); i++) {
        /* Rzutowanie na unsigned char — OBOWIĄZKOWE dla ctype.h */
        unsigned char c = (unsigned char)tekst[i];

        if (isalpha(c))      liter++;   /* litera */
        else if (isdigit(c)) cyfr++;    /* cyfra */
        else if (isspace(c)) spacji++;  /* biały znak */
        else                 innych++;  /* reszta: !, , */
    }

    printf("Liter: %d, Cyfr: %d, Spacji: %d, Innych: %d\n",
           liter, cyfr, spacji, innych);

    /* --- Konwersja na wielkie litery --- */
    char kopia[50];
    strncpy(kopia, tekst, sizeof(kopia) - 1);
    kopia[sizeof(kopia) - 1] = '\0';

    for (size_t i = 0; i < strlen(kopia); i++) {
        kopia[i] = (char)toupper((unsigned char)kopia[i]);
    }
    printf("Uppercase: \"%s\"\n", kopia);

    /* --- Konwersja na małe litery --- */
    for (size_t i = 0; i < strlen(kopia); i++) {
        kopia[i] = (char)tolower((unsigned char)kopia[i]);
    }
    printf("Lowercase: \"%s\"\n", kopia);

    /* --- Sprawdzanie poszczególnych znaków --- */
    unsigned char testowe[] = {'A', '5', ' ', '!', '\n', '\t'};
    const char *nazwy[]     = {"'A'", "'5'", "' '", "'!'", "'\\n'", "'\\t'"};

    printf("\n%-6s  %-6s %-6s %-6s %-6s %-6s %-6s\n",
           "Znak","alpha","digit","space","upper","lower","print");

    for (int i = 0; i < 6; i++) {
        unsigned char c = testowe[i];
        printf("%-6s  %-6s %-6s %-6s %-6s %-6s %-6s\n",
               nazwy[i],
               isalpha(c) ? "TAK" : "NIE",
               isdigit(c) ? "TAK" : "NIE",
               isspace(c) ? "TAK" : "NIE",
               isupper(c) ? "TAK" : "NIE",
               islower(c) ? "TAK" : "NIE",
               isprint(c) ? "TAK" : "NIE");
    }

    return 0;
}
```

|------|--------|------------|

## 13. Kodowania znaków — ASCII, EBCDIC, Unicode, UTF-8, UTF-16, UTF-32

### Teoria

**Kodowanie znaków** to przyporządkowanie liczb całkowitych (kodów) do znaków.

**ASCII** (American Standard Code for Information Interchange):
- Zakres: 0–127 (7 bitów)
- Kody 0–31 i 127: znaki kontrolne (LF, CR, TAB, BEL, ...)
- Kody 32–126: znaki drukowalne (spacje, litery, cyfry, interpunkcja)
- Kluczowe wartości: `'0'` = 48, `'A'` = 65, `'a'` = 97
- Dzięki tej regularności można konwertować: `cyfra - '0'` daje wartość numeryczną, `litera - 'a'` daje pozycję w alfabecie

**EBCDIC** (Extended Binary Coded Decimal Interchange Code):
- Kodowanie IBM dla mainframe'ów
- Niekompatybilne z ASCII (inne kody dla liter i cyfr)
- Rzadko spotykane poza środowiskami IBM z/OS

**Unicode:**
- Próba objęcia wszystkich znaków świata jednym standardem
- Zakres kodów: U+0000 do U+10FFFF (ponad 1 milion możliwych znaków)
- Notacja: `U+XXXX` (hex) — np. `U+0041` = 'A', `U+00F3` = 'ó'
- Zgodny z ASCII dla pierwszych 128 znaków
- Samo Unicode nie mówi jak przechować kody w pamięci — do tego służą kodowania UTF

**UTF-8:**
- Zmienna długość: 1–4 bajty na znak
- ASCII (U+0000–U+007F): 1 bajt, bit 0 na początku → kompatybilność wsteczna z ASCII
- U+0080–U+07FF: 2 bajty (`110xxxxx 10xxxxxx`)
- U+0800–U+FFFF: 3 bajty (`1110xxxx 10xxxxxx 10xxxxxx`)
- U+10000–U+10FFFF: 4 bajty (`11110xxx 10xxxxxx 10xxxxxx 10xxxxxx`)
- Zasada: tylko najkrótszy możliwy kod jest poprawny
- Dominujące kodowanie w internecie i Linuksie

**UTF-16:**
- Zmienna długość: 2 lub 4 bajty na znak
- BMP (U+0000–U+D7FF, U+E000–U+FFFF): 2 bajty
- Powyżej U+10000: 4 bajty jako **para zastępcza** (surrogate pair): `D800₁₆ + 10 bitów` + `DC00₁₆ + 10 bitów`
- Używany wewnętrznie przez Windows (WinAPI) i Java

**UTF-32:**
- Stała długość: 4 bajty na każdy znak
- Proste indeksowanie, ale duże pliki
- Używany wewnętrznie przez Python 3, GCC

### Kod z komentarzami

```c
#include <stdio.h>
#include <string.h>   /* strlen */

int main(void) {
    /* --- Właściwości ASCII --- */
    printf("=== Kody ASCII ===\n");

    /* Różnica między kodem cyfry a wartością numeryczną */
    char cyfra = '7';
    int wartosc = cyfra - '0';   /* '7' (55) - '0' (48) = 7 */
    printf("'7' - '0' = %d\n", wartosc);

    /* Różnica między dużą a małą literą */
    char duza  = 'A';            /* kod 65 */
    char mala  = duza + 32;     /* 65 + 32 = 97 = 'a' */
    printf("'A' + 32 = '%c' (kod %d)\n", mala, mala);

    /* Wypisanie kodów ASCII dla liter */
    printf("\nKody ASCII: ");
    for (char c = 'A'; c <= 'E'; c++) {
        printf("%c=%d ", c, (int)c);   /* A=65 B=66 C=67 D=68 E=69 */
    }
    printf("\n");

    /* Konwersja pozycji w alfabecie */
    char litera = 'D';
    int pozycja = litera - 'A';   /* 0-indeksowana pozycja: D=3 */
    printf("Pozycja '%c' w alfabecie: %d (od 0)\n", litera, pozycja);

    /* --- UTF-8: łańcuch z polskimi znakami --- */
    /* Zakładamy że plik źródłowy jest w UTF-8 */
    printf("\n=== UTF-8 ===\n");

    const char *polski = "ąęó";   /* 3 polskie znaki — każdy 2 bajty w UTF-8 */

    /* strlen liczy bajty, nie znaki Unicode! */
    printf("Tekst: \"%s\"\n", polski);
    printf("strlen (bajty): %zu\n", strlen(polski));   /* 6 (3 x 2 bajty) */
    printf("Znaki Unicode:  3 (każdy zajmuje 2 bajty w UTF-8)\n");

    /* Wyświetlamy bajty UTF-8 dla 'ą' (U+0105) */
    /* UTF-8: 11000100 10000101 = 0xC4 0x85 */
    printf("\nBajty UTF-8 dla 'ą' (U+0105):\n");
    unsigned char a_ogonek[] = {0xC4, 0x85, 0x00};
    for (int i = 0; a_ogonek[i] != 0; i++) {
        printf("  Bajt %d: 0x%02X (%d)\n", i, a_ogonek[i], a_ogonek[i]);
    }

    /* --- Sprawdzenie kodu znaku --- */
    printf("\n=== Kody wybranych znaków ASCII ===\n");
    printf("Spacja ' ' = %d\n", (int)' ');     /* 32 */
    printf("'0'        = %d\n", (int)'0');     /* 48 */
    printf("'A'        = %d\n", (int)'A');     /* 65 */
    printf("'a'        = %d\n", (int)'a');     /* 97 */
    printf("DEL        = %d\n", 127);
    printf("EOF (makro)= %d\n", EOF);          /* -1 */

    /* --- Tablica ASCII 32-126 --- */
    printf("\n=== Drukowalne znaki ASCII (32-126) ===\n");
    for (int i = 32; i <= 126; i++) {
        printf("%3d='%c' ", i, i);
        if ((i - 32 + 1) % 8 == 0) printf("\n");   /* 8 na wiersz */
    }
    printf("\n");

    return 0;
}
```

## 14. Typy łańcuchowe i ich rozmiary — char, char8_t, char16_t, char32_t, wchar_t

### Teoria

Standardowy C używa `char` do przechowywania znaków, ale jeden `char` to 1 bajt — niewystarczający dla znaków Unicode spoza ASCII.

| Typ | Nagłówek | Przedrostek literału | Kodowanie | Rozmiar |
|-----|----------|---------------------|-----------|---------|
| `char` | brak | brak | implementacja (ASCII/UTF-8) | 1 bajt |
| `char8_t` | `<uchar.h>` | `u8` | UTF-8 | 1 bajt (od C23) |
| `char16_t` | `<uchar.h>` | `u` | UTF-16 | 2 bajty |
| `char32_t` | `<uchar.h>` | `U` | UTF-32 | 4 bajty |
| `wchar_t` | `<wchar.h>` | `L` | implementacja (UTF-16 lub UTF-32) | zwykle 2 lub 4 bajty |

Ważna różnica między `sizeof` a `strlen`:
- `sizeof(tablica)` — rozmiar całej tablicy w bajtach (łącznie z `'\0'` i pustym miejscem)
- `strlen(tablica)` — liczba znaków **przed** `'\0'`

### Kod z komentarzami

```c
#include <stdio.h>
#include <string.h>   /* strlen */
#include <wchar.h>    /* wchar_t, wprintf, wcslen */
#include <stdlib.h>

int main(void) {
    /* --- char — klasyczny łańcuch --- */
    char nazwa[] = "Wielka Niedzwiedzica";   /* +'\0' automatycznie */
    char ktos[]  = {'M', 'i', 'k', 'o', 'l', 'a', 'j', '\0'};
    char imie[20];   /* niezainicjalizowany bufor na 20 znaków */

    printf("=== char ===\n");
    printf("nazwa   = \"%s\"\n",     nazwa);
    printf("sizeof(nazwa)  = %zu\n", sizeof(nazwa));  /* 21 (20 + '\0') */
    printf("strlen(nazwa)  = %zu\n", strlen(nazwa));  /* 20 */

    printf("ktos    = \"%s\"\n",     ktos);
    printf("sizeof(ktos)   = %zu\n", sizeof(ktos));   /* 8 (7 + '\0') */
    printf("strlen(ktos)   = %zu\n", strlen(ktos));   /* 7 */
    printf("sizeof != strlen: bufor vs. zawartość\n\n");

    /* sizeof(imie) = 20, ale strlen(imie) jest niezdefiniowane — niezinicjalizowana */
    /* Inicjalizacja: */
    imie[0] = '\0';    /* pusty łańcuch */
    printf("imie (pusty): strlen = %zu\n\n", strlen(imie));  /* 0 */

    /* --- wchar_t — szerokie znaki (wide characters) --- */
    wchar_t wlancuch[] = L"Zażółć gęślą jaźń";  /* L = wide string literal */

    printf("=== wchar_t ===\n");

    /* wcslen liczy znaki wchar_t, a nie bajty */
    printf("wcslen(L\"Zażółć...\") = %zu znaków\n", wcslen(wlancuch));
    printf("sizeof(wchar_t) = %zu bajtow\n", sizeof(wchar_t));

    /* wprintf wypisuje wchar_t na stdout */
    /* Wymagane: ustawienie locale żeby terminal obsługiwał UTF */
    /* setlocale(LC_ALL, ""); — pominięte dla prostoty */
    /* wprintf(L"Tekst: %ls\n", wlancuch); */

    /* --- Literały dla różnych typów --- */
    printf("\n=== Rozmiary typów znakowych ===\n");
    printf("sizeof(char)     = %zu\n", sizeof(char));      /* 1 */
    printf("sizeof(wchar_t)  = %zu\n", sizeof(wchar_t));   /* 2 lub 4 */

    /* char16_t i char32_t wymagają C11 */
    /* #include <uchar.h> */
    /* char16_t u16[] = u"Hello"; */
    /* char32_t u32[] = U"Hello"; */

    /* --- Arytmetyka na znakach --- */
    printf("\n=== Arytmetyka na znakach ===\n");

    /* Konwersja znaku cyfry na wartość liczbową */
    char c_cyfra = '7';
    int  wartosc = c_cyfra - '0';
    printf("'%c' - '0' = %d\n", c_cyfra, wartosc);  /* 7 */

    /* Konwersja znaku litery na pozycję w alfabecie */
    char c_litera = 'g';
    int  pozycja  = c_litera - 'a';
    printf("'%c' - 'a' = %d (0-indeksowane)\n", c_litera, pozycja);  /* 6 */

    /* Iteracja po znakach przez arytmetykę */
    printf("Litery: ");
    for (char c = 'A'; c <= 'Z'; c++) {
        printf("%c", c);
    }
    printf("\n");

    return 0;
}
```

## 15. Zarządzanie pamięcią — stos, sterta, pamięć statyczna

### Teoria

W programach C pamięć dzielona jest na kilka obszarów:

**Stos (ang. stack):**
- Strukturę FILO (First In, Last Out) — elementy odkładane i zdejmowane w odwrotnej kolejności
- Przechowuje: zmienne lokalne, parametry funkcji, adresy powrotu
- Alokacja i dealokacja **automatyczna** — przy wejściu/wyjściu z bloku `{}`
- Rozmiar ograniczony (typowo 1–8 MB — zależy od systemu)
- Brak fragmentacji — wskaźnik stosu przesuwa się liniowo
- Brak możliwości relokacji — adres zmiennej lokalnej nie może się zmienić

**Sterta (ang. heap):**
- Zarządzana ręcznie: `malloc`, `calloc`, `realloc`, `free`
- Lub automatycznie przez Garbage Collector (w C nie ma GC!)
- Możliwość zmiany rozmiaru (`realloc`)
- Może wystąpić **fragmentacja** — wolna pamięć w wielu małych kawałkach
- Nieograniczony (przez dostępną pamięć systemu)

**Pamięć statyczna (ang. static storage):**
- Zmienne globalne i zmienne `static`
- Istnieją przez cały czas życia programu
- Inicjalizowane zerem domyślnie

**Segment kodu (text):**
- Skompilowany kod programu
- Zwykle tylko do odczytu

```
+-----------------+
|   Stack (stos)  |  ← rośnie w dół
|      ...        |
+-----------------+
|   (wolne)       |
+-----------------+
|   Heap (sterta) |  ← rośnie w górę
|      ...        |
+-----------------+
|   BSS (zerowane)|  zmienne globalne niezainicjalizowane
+-----------------+
|   Data          |  zmienne globalne zainicjalizowane
+-----------------+
|   Text (kod)    |
+-----------------+
```

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>   /* malloc, calloc, realloc, free */
#include <string.h>   /* memset */

/* Zmienna globalna — pamięć statyczna, istnieje przez cały program */
int licznik_globalny = 0;   /* inicjalizowana, w segmencie Data */
int zmienna_bss;            /* niezainicjalizowana, w segmencie BSS, = 0 */

void funkcja_ze_stosem(void) {
    /* Ta zmienna siedzi na stosie — znika po wyjściu z funkcji */
    int lokalna = 42;
    printf("Stos: lokalna = %d, adres = %p\n", lokalna, (void*)&lokalna);
    /* Po powrocie z funkcji pamięć dla 'lokalna' jest automatycznie zwalniana */
}

void funkcja_ze_statica(void) {
    /* static zmienia klasę pamięci z auto (stos) na statyczną */
    /* Zmienna istnieje między wywołaniami — nie jest tworzona od nowa */
    static int licznik = 0;   /* inicjalizowana tylko raz, przy pierwszym wywołaniu */
    licznik++;
    printf("Static: licznik = %d (zachowany między wywołaniami)\n", licznik);
}

int main(void) {
    printf("=== Stos ===\n");
    funkcja_ze_stosem();
    funkcja_ze_stosem();   /* ta sama przestrzeń adresowa na stosie — może ten sam adres */

    printf("\n=== Pamięć statyczna ===\n");
    printf("Globalna: licznik_globalny = %d\n", licznik_globalny);
    printf("BSS (globalna niezinit.): zmienna_bss = %d\n", zmienna_bss);  /* 0 */

    funkcja_ze_statica();
    funkcja_ze_statica();
    funkcja_ze_statica();

    printf("\n=== Sterta (heap) — malloc / free ===\n");

    /* malloc: alokuje n bajtów, zwraca void* lub NULL przy błędzie */
    int *tablica = (int*)malloc(5 * sizeof(int));
    if (tablica == NULL) {
        perror("malloc");
        exit(EXIT_FAILURE);
    }
    printf("Sterta: tablica pod adresem %p\n", (void*)tablica);

    /* Inicjalizacja (malloc NIE zeruje pamięci) */
    for (int i = 0; i < 5; i++) {
        tablica[i] = (i + 1) * 10;   /* 10, 20, 30, 40, 50 */
    }
    printf("Wartości: ");
    for (int i = 0; i < 5; i++) printf("%d ", tablica[i]);
    printf("\n");

    /* realloc: zmiana rozmiaru istniejącego bloku */
    /* Może przenieść blok pod nowy adres! Stary wskaźnik staje się nieważny */
    int *wiekszy = (int*)realloc(tablica, 8 * sizeof(int));
    if (wiekszy == NULL) {
        perror("realloc");
        free(tablica);   /* realloc nie zwolnił oryginału przy błędzie */
        exit(EXIT_FAILURE);
    }
    tablica = wiekszy;   /* aktualizujemy wskaźnik — stary może być nieważny */

    /* Nowe elementy nie są zainicjalizowane */
    for (int i = 5; i < 8; i++) {
        tablica[i] = (i + 1) * 10;
    }
    printf("Po realloc (8 el.): ");
    for (int i = 0; i < 8; i++) printf("%d ", tablica[i]);
    printf("\n");

    /* calloc: alokuje i zeruje pamięć */
    double *zera = (double*)calloc(4, sizeof(double));
    if (!zera) { perror("calloc"); free(tablica); exit(1); }
    printf("calloc zeruje: ");
    for (int i = 0; i < 4; i++) printf("%.1f ", zera[i]);   /* 0.0 0.0 0.0 0.0 */
    printf("\n");

    /* free: zwolnienie pamięci — OBOWIĄZKOWE dla każdego malloc/calloc/realloc */
    free(tablica);
    tablica = NULL;   /* dobra praktyka: NULL po free, zapobiega double-free */

    free(zera);
    zera = NULL;

    printf("Pamięć zwolniona\n");

    return 0;
}
```

### Co program wypisze

```
=== Stos ===
Stos: lokalna = 42, adres = 0x7ffd...
Stos: lokalna = 42, adres = 0x7ffd...

=== Pamięć statyczna ===
Globalna: licznik_globalny = 0
BSS (globalna niezinit.): zmienna_bss = 0
Static: licznik = 1 (zachowany między wywołaniami)
Static: licznik = 2 (zachowany między wywołaniami)
Static: licznik = 3 (zachowany między wywołaniami)

=== Sterta (heap) — malloc / free ===
Sterta: tablica pod adresem 0x55a3...
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| Brak sprawdzenia `malloc == NULL` | Crash (segfault) przy próbie dereferencji | Zawsze `if (ptr == NULL)` |
| Brak `free` | Wyciek pamięci | Każdy `malloc/calloc/realloc` musi mieć `free` |
| `free` dwa razy (double-free) | UB — uszkodzenie sterty | Po `free` ustaw `ptr = NULL`; `free(NULL)` jest bezpieczne |
| Użycie wskaźnika po `realloc` | Stary wskaźnik może wskazywać zwolnioną pamięć | Zawsze `ptr = realloc(ptr, n); if (!ptr) ...` |
| Zwracanie wskaźnika do zmiennej lokalnej | Wskazuje na pamięć po wyjściu z funkcji (dangling pointer) | Alokuj dynamicznie lub przekaż bufor jako parametr |

## 16. Specyfikatory klas pamięci — static, extern, register, constexpr, thread_local

### Teoria

**Klasa pamięci** określa, gdzie zmienna jest przechowywana, jak długo żyje i jaka jest jej widzialność (linkage).

| Specyfikator | Gdzie przechowywana | Czas życia | Widzialność |
|-------------|---------------------|------------|-------------|
| (brak) / `auto` | stos | blok | lokalny blok |
| `static` (w funkcji) | pamięć statyczna | cały program | lokalny blok |
| `static` (poza funkcją) | pamięć statyczna | cały program | tylko bieżący plik |
| `extern` | inny plik (deklaracja) | cały program | wiele plików |
| `register` | rejestr CPU (sugestia) | blok | lokalny blok |
| `constexpr` (C23) | zależy | kompilacja | bieżący zakres |
| `thread_local` (C11) | osobno dla każdego wątku | wątek | zależy |

**`auto`** — historyczny specyfikator (C89), dziś zbędny; zmienne lokalne są auto domyślnie. W C++ `auto` ma inne znaczenie.

**`static` wewnątrz funkcji** — zmienna zachowuje wartość między wywołaniami; inicjalizowana tylko raz (przy pierwszym wywołaniu).

**`static` poza funkcją** — ogranicza widzialność do pliku `.c` (internal linkage). Ukrywa przed innymi jednostkami kompilacji.

**`extern`** — deklaracja (nie definicja) zmiennej zdefiniowanej w innym pliku. Mówi kompilatorowi: "ta zmienna istnieje, szukaj jej w innym pliku".

**`register`** — sugestia dla kompilatora, by przechował zmienną w rejestrze CPU (nowoczesne kompilatory ignorują tę sugestię). Efekt: nie można brać adresu (`&`) zmiennej `register`.

**`constexpr`** (od C23) — wartość ustalana w czasie kompilacji, podobne do `const`, ale z gwarancją stałości kompilacyjnej.

**`thread_local`** (od C11, nagłówek `<threads.h>`) — każdy wątek ma własną kopię zmiennej.

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>

/* static poza funkcją — widoczne tylko w tym pliku */
static int licznik_pliku = 0;

/* Bez static — widoczne z innych plików (external linkage) */
int zmienna_globalna = 100;

/* Funkcja demonstrująca static wewnątrz funkcji */
void licznik_wywolan(void) {
    /* static: inicjalizowana raz, wartość zachowana między wywołaniami */
    static int wywolania = 0;
    wywolania++;   /* inkrementacja zachowana między wywołaniami */
    printf("  Wywołanie nr: %d\n", wywolania);
}

/* Funkcja demonstrująca auto (domyślna klasa lokalna) */
void auto_demo(void) {
    /* int x; to dokładnie to samo co auto int x; */
    /* auto int x = 10; */ /* archaiczne, nie pisz tak */
    int x = 10;   /* zmienna lokalna na stosie, znika po wyjściu z funkcji */
    printf("  auto x = %d, adres = %p\n", x, (void*)&x);
}

/* Funkcja demonstrująca register */
long suma_tablicy(const int *arr, int n) {
    /* register: sugestia dla kompilatora, by użył rejestru CPU */
    /* Nowoczesne kompilatory zwykle to i tak robią automatycznie */
    register long suma = 0;
    /* register int i — nie możemy brać adresu: &i byłoby błędem kompilacji */
    for (int i = 0; i < n; i++) {
        suma += arr[i];
    }
    return suma;
}

int main(void) {
    printf("=== static w funkcji ===\n");
    licznik_wywolan();   /* Wywołanie nr: 1 */
    licznik_wywolan();   /* Wywołanie nr: 2 */
    licznik_wywolan();   /* Wywołanie nr: 3 */

    printf("\n=== static poza funkcją (plik) ===\n");
    licznik_pliku = 42;
    printf("  licznik_pliku = %d (widoczny tylko w tym pliku)\n", licznik_pliku);

    printf("\n=== extern ===\n");
    /* extern int zmienna_globalna; — deklaracja, gdyby była w innym pliku */
    /* Tutaj zmienna_globalna jest w tym samym pliku, więc extern nie potrzebne */
    printf("  zmienna_globalna = %d\n", zmienna_globalna);

    printf("\n=== auto (lokalna) ===\n");
    auto_demo();
    auto_demo();   /* może ten sam adres — ten sam frame stosu */

    printf("\n=== register ===\n");
    int dane[] = {1, 2, 3, 4, 5};
    long s = suma_tablicy(dane, 5);
    printf("  Suma = %ld\n", s);   /* 15 */

    /* --- constexpr (C23) --- */
    /* Wartość ustalona w czasie kompilacji */
    /* constexpr int MAX = 100; */   /* wymaga kompilatora C23 */
    /* Odpowiednik w starszym C: */
    enum { MAX_ROZMIAR = 100 };        /* stała kompilacyjna przez enum */
    static const int BUFSIZ_LOKALNY = 256;   /* const w pamięci statycznej */

    printf("\n=== Stałe kompilacyjne ===\n");
    printf("  MAX_ROZMIAR (enum) = %d\n",     MAX_ROZMIAR);
    printf("  BUFSIZ_LOKALNY (const) = %d\n", BUFSIZ_LOKALNY);

    return 0;
}
```

### Co program wypisze

```
=== static w funkcji ===
  Wywołanie nr: 1
  Wywołanie nr: 2
  Wywołanie nr: 3

=== static poza funkcją (plik) ===
  licznik_pliku = 42 (widoczny tylko w tym pliku)

=== extern ===
  zmienna_globalna = 100

=== auto (lokalna) ===
  auto x = 10, adres = 0x7ffd...
  auto x = 10, adres = 0x7ffd...

=== register ===
  Suma = 15

=== Stałe kompilacyjne ===
  MAX_ROZMIAR (enum) = 100
  BUFSIZ_LOKALNY (const) = 256
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| Wielokrotna definicja zmiennej `extern` (zamiast deklaracji) | Błąd linkera | W `.h` piszemy `extern int x;` (deklaracja), w jednym `.c` `int x = 0;` (definicja) |
| `static` lokalne bez inicjalizacji (w intencji resetu) | Wartość zachowana z poprzedniego wywołania | Inicjalizuj jawnie lub użyj zmiennej auto |
| `&` na zmiennej `register` | Błąd kompilacji | Nie bierz adresu zmiennych `register` |
| Używanie zmiennych `thread_local` bez synchronizacji | Każdy wątek ma własną kopię — zmiany niewidoczne dla innych | To jest celowe — `thread_local` to izolacja na poziomie wątku |

## 17. Ponowne otwarcie strumienia — freopen

### Teoria

**`freopen`** zamyka istniejący strumień i otwiera nowy (lub ten sam plik w innym trybie) za pomocą tego samego wskaźnika `FILE*`. Szczególnie użyteczne do **przekierowania standardowych strumieni** (`stdin`, `stdout`, `stderr`) — np. żeby program pisał do pliku zamiast na ekran, bez zmiany kodu używającego `printf`.

Sygnatury:
```c
FILE *freopen(const char *nazwa, const char *tryb, FILE *strumien);
```

Jeśli `nazwa` to `NULL`, próbuje otworzyć ten sam plik w nowym trybie (zachowanie zależy od platformy).

Typowe zastosowania:
- Przekierowanie `stderr` do pliku logów
- Przekierowanie `stdin` z pliku (wczytywanie testowych danych bez zmiany `scanf`)
- Przekierowanie `stdout` do pliku z poziomu programu

### Kod z komentarzami

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    /* --- Przekierowanie stdout do pliku --- */
    printf("Ta linia idzie na EKRAN (stdout)\n");

    /* freopen przekierowuje stdout do pliku "wyjscie.txt" */
    FILE *fp = freopen("wyjscie.txt", "w", stdout);
    if (fp == NULL) {
        /* Błąd — stderr nadal działa, bo nie przekierowaliśmy */
        fprintf(stderr, "Blad freopen dla stdout!\n");
        exit(EXIT_FAILURE);
    }

    /* Od teraz printf pisze do pliku, nie na ekran */
    printf("Ta linia idzie do PLIKU wyjscie.txt\n");
    printf("Liczba: %d\n", 42);
    printf("PI ~ %.4f\n", 3.14159);

    /* Zamknij przekierowanie — przywróć stdout do terminala */
    fclose(stdout);

    /* UWAGA: po fclose(stdout) nie możemy już pisać na stdout! */
    /* Poniższe trafi na stderr (który nadal działa) */
    fprintf(stderr, "Stdout zamkniety — ta linia idzie na STDERR\n");

    /* --- Przekierowanie stderr do pliku logów --- */
    /* To jest bardzo popularne w aplikacjach produkcyjnych */
    FILE *log_fp = freopen("aplikacja.log", "a", stderr);
    if (log_fp == NULL) {
        /* Nie możemy nic zrobić — stderr też nie działa */
        exit(EXIT_FAILURE);
    }

    fprintf(stderr, "[LOG] Program uruchomiony\n");
    fprintf(stderr, "[LOG] Wersja 1.0\n");
    fprintf(stderr, "[LOG] Blad: cos poszlo nie tak (przykladowy log)\n");

    /* --- Przekierowanie stdin z pliku (symulacja wejścia testowego) --- */
    /* Tworzymy plik testowy */
    FILE *testinput = fopen("testinput.txt", "w");
    if (testinput) {
        fprintf(testinput, "25\n");   /* symulowane wejście użytkownika */
        fprintf(testinput, "Kowalski\n");
        fclose(testinput);
    }

    if (freopen("testinput.txt", "r", stdin) == NULL) {
        /* Nie możemy przekierować stdin */
        exit(EXIT_FAILURE);
    }

    int wiek;
    char nazwisko[50];

    /* Te scanf czytają teraz z pliku, nie z klawiatury */
    if (scanf("%d", &wiek) == 1) {
        /* wiek = 25 (z pliku) */
    }
    /* Czyszczenie bufora po scanf */
    while (getchar() != '\n');

    if (fgets(nazwisko, sizeof(nazwisko), stdin) != NULL) {
        /* Usuwamy '\n' */
        for (int i = 0; nazwisko[i]; i++) {
            if (nazwisko[i] == '\n') { nazwisko[i] = '\0'; break; }
        }
    }

    fprintf(stderr, "[LOG] Wczytano z pliku: wiek=%d, nazwisko=%s\n",
            wiek, nazwisko);

    return 0;
}
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |

## 18. Tokeny — strtok

### Teoria

**`strtok`** dzieli łańcuch znaków na tokeny — fragmenty oddzielone separatorami. Modyfikuje oryginalny łańcuch (zastępuje separatory `'\0'`), dlatego nie wolno jej stosować na literałach ani `const char*`.

```c
char *strtok(char *s1, const char *s2);
```

Pierwsze wywołanie: przekazujemy `s1` = łańcuch do podziału, `s2` = zbiór separatorów. Zwraca wskaźnik na pierwszy token.

Kolejne wywołania: przekazujemy `s1 = NULL` — `strtok` pamięta wewnętrznie (w statycznej zmiennej) gdzie skończyła. Zwraca kolejne tokeny lub `NULL` gdy skończyły się tokeny.

**Uwaga:** `strtok` używa statycznej zmiennej globalnej — nie jest bezpieczna wątkowo i nie można jej zagnieżdżać (dwie różne pętle `strtok` na raz). Wątkowo bezpieczna wersja POSIX: **`strtok_r`**, a bezpieczna wersja standardowa C11: **`strtok_s`**.

Po podziale oryginalny łańcuch wygląda: `"Ala\0ma\0kota\0i\0psa\0"` — pierwsze `','`, `';'` i spacje zastępowane przez `'\0'`.

### Kod z komentarzami

```c
#include <stdio.h>
#include <string.h>

int main(void) {
    /* --- Podstawowe użycie strtok --- */
    char tekst[] = "Ala,ma;kota i psa";  /* MUSI być modyfikowalna tablica */
    /* const char *tekst = "..."; — BŁĄD! strtok modyfikuje łańcuch */

    const char *separatory = ",; ";   /* separatory: przecinek, średnik, spacja */

    printf("Oryginalny: \"%s\"\n", tekst);

    /* Pierwsze wywołanie: podajemy łańcuch */
    char *token = strtok(tekst, separatory);

    int numer = 1;
    while (token != NULL) {
        printf("  Token %d: [%s]\n", numer++, token);
        /* Kolejne wywołania: NULL zamiast łańcucha */
        token = strtok(NULL, separatory);
    }

    /* Uwaga: tekst jest teraz "Ala\0ma\0kota\0i\0psa" */
    /* Nie możemy go wypisać przez printf("%s", tekst) jako całości */

    /* --- Praktyczny przykład: parsing CSV --- */
    printf("\n--- Parsowanie CSV ---\n");
    char csv[] = "Ala;22;85.5";   /* imię;wiek;wynik */

    char *imie   = strtok(csv, ";");
    char *s_wiek = strtok(NULL, ";");
    char *s_wyn  = strtok(NULL, ";");

    if (imie && s_wiek && s_wyn) {
        int    wiek  = atoi(s_wiek);      /* łańcuch → int */
        double wynik = atof(s_wyn);       /* łańcuch → double */
        printf("Imię: %s, Wiek: %d, Wynik: %.1f\n", imie, wiek, wynik);
    }

    /* --- Parsowanie ścieżki do pliku --- */
    printf("\n--- Parsowanie ścieżki ---\n");
    char sciezka[] = "/home/user/dokumenty/plik.txt";

    char *skladnik = strtok(sciezka, "/");
    while (skladnik != NULL) {
        printf("  Składnik: [%s]\n", skladnik);
        skladnik = strtok(NULL, "/");
    }

    /* --- Parsowanie wielu separatorów pod rząd (są pomijane) --- */
    printf("\n--- Wielokrotne separatory (ignorowane) ---\n");
    char wielosep[] = "jeden,,,,dwa   trzy;;cztery";
    char *tok = strtok(wielosep, ",; ");

    while (tok != NULL) {
        printf("  [%s]\n", tok);
        tok = strtok(NULL, ",; ");
    }

    return 0;
}
```

### Co program wypisze

```
Oryginalny: "Ala,ma;kota i psa"
  Token 1: [Ala]
  Token 2: [ma]
  Token 3: [kota]
  Token 4: [i]
  Token 5: [psa]

--- Parsowanie CSV ---
Imię: Ala, Wiek: 22, Wynik: 85.5

--- Parsowanie ścieżki ---
  Składnik: [home]
  Składnik: [user]
  Składnik: [dokumenty]
  Składnik: [plik.txt]

--- Wielokrotne separatory (ignorowane) ---
  [jeden]
  [dwa]
  [trzy]
  [cztery]
```

### Typowe błędy

| Błąd | Skutek | Jak unikać |
|------|--------|------------|
| `strtok` na literale string: `strtok("Ala,ma", ",")` | Modyfikacja stałego obszaru pamięci — crash lub UB | Zawsze kopiuj do tablicy: `char buf[] = "..."; strtok(buf, ...)` |
| Zagnieżdżone pętle `strtok` | Wewnętrzna statyczna zmienna się myli — błędny podział | Użyj `strtok_r` (POSIX) lub `strtok_s` (C11) |
| Zakładanie że oryginał jest niezmieniony po `strtok` | Separatory zastąpione przez `'\0'` | Jeśli potrzebujesz oryginał, zrób kopię przed `strtok` |
| Pobieranie tokenów po skończeniu pętli | `strtok` zwróciło `NULL` — dalsze wywołania z `NULL` mogą zwrócić NULL lub cokolwiek | Przerwij pętlę gdy `token == NULL` |

## Podsumowanie kluczowych zasad

1. **Zawsze sprawdzaj wynik `fopen`** — null pointer to crash.
2. **Zawsze zamykaj pliki `fclose`** — niezamknięty plik to utrata danych i wyciek zasobów.
3. **Zmienna na wynik `fgetc` musi być `int`** — `char` nie pomieści `EOF`.
4. **Używaj `fgets` zamiast `gets`** — `gets` jest niebezpieczna i usunięta w C11.
5. **`fflush(stdin)` jest UB** — używaj `while(getchar() != '\n');`.
6. **Pliki binarne otwieraj z `b`** — unikniesz problemów z `\r\n` na Windows.
7. **`strtok` modyfikuje łańcuch** — nie używaj na literałach ani `const`.
8. **`sizeof` ≠ `strlen`** — pierwsze podaje rozmiar bufora, drugie długość tekstu.
9. **`strdup` wymaga `free`** — każda alokacja musi mieć dealokację.
10. **Rzutuj na `unsigned char`** przy funkcjach `ctype.h** — unikniesz UB z ujemnymi kodami.
