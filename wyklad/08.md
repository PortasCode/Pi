## 1. fprintf i printf – formatowanie wyjścia

### Teoria

`printf` to funkcja drukująca sformatowany tekst na standardowe wyjście (terminal). `fprintf` działa identycznie, ale pozwala wskazać dowolny strumień – np. plik lub `stderr`. Obie funkcje przyjmują łańcuch formatujący, w którym znaki procenta `%` oznaczają miejsca wstawienia kolejnych argumentów.

Format każdego specyfikatora wygląda tak:

```
%[flagi][szerokość_pola][.precyzja][rozmiar]specyfikator
```

**Flagi:**
- `+` – zawsze drukuje znak liczby (+ lub -)
- `-` – wyrównanie do lewej w polu
- `spacja` – wstawia spację przed liczbą dodatnią
- `0` – dopełnia zerami zamiast spacjami
- `#` – prefiks dla liczb ósemkowych (`0`) i szesnastkowych (`0x`)

**Specyfikatory (najważniejsze):**
- `d`, `i` – liczba całkowita ze znakiem
- `u` – liczba całkowita bez znaku
- `f`, `F` – liczba zmiennoprzecinkowa (stały zapis dziesiętny)
- `e`, `E` – zapis naukowy (np. `1.23e+04`)
- `g`, `G` – automatycznie wybiera `f` lub `e`
- `x`, `X` – szesnastkowy
- `o` – ósemkowy
- `b`, `B` – binarny (od C23)
- `c` – pojedynczy znak
- `s` – łańcuch znaków
- `p` – adres wskaźnika
- `n` – zapisuje do zmiennej ile znaków zostało wydrukowanych (rzadko używany, niebezpieczny)

**Modyfikatory rozmiaru:**
- `hh` – `char`
- `h` – `short`
- `l` – `long`
- `ll` – `long long`
- `L` – `long double`

### Kompletny działający kod

```c
#include <stdio.h>   /* potrzebne dla printf i fprintf */
#include <stdlib.h>  /* potrzebne dla EXIT_SUCCESS */

int main(void) {

    /* --- Podstawowe typy --- */
    int ilosc = 5;           /* liczba całkowita */
    float cena = 12.99f;     /* liczba zmiennoprzecinkowa */
    double pi = 3.14159265;  /* podwójna precyzja */
    char litera = 'A';       /* pojedynczy znak */
    char tekst[] = "witaj";  /* łańcuch znaków */

    /* %d – liczba całkowita dziesiętna */
    printf("Ilosc: %d\n", ilosc);

    /* %.2f – liczba zmiennoprzecinkowa z 2 miejscami po przecinku */
    printf("Cena: %.2f PLN\n", cena);

    /* %3d – pole szerokości 3, wyrównanie do prawej */
    printf("Zakupiono %3d sztuki po %.2f PLN.\n", ilosc, cena);

    /* %03d – dopełnienie zerami do szerokości 3 */
    printf("Numer: %03d\n", 7);

    /* %-10s – łańcuch wyrównany do lewej w polu 10 znaków */
    printf("|%-10s|\n", tekst);

    /* %10s – łańcuch wyrównany do prawej w polu 10 znaków */
    printf("|%10s|\n", tekst);

    /* %c – pojedynczy znak */
    printf("Litera: %c\n", litera);

    /* %x – szesnastkowy zapis liczby */
    printf("255 szesnastkowo: %x\n", 255);

    /* %#x – z prefiksem 0x */
    printf("255 szesnastkowo z prefiksem: %#x\n", 255);

    /* %o – ósemkowy zapis liczby */
    printf("255 osemkowo: %o\n", 255);

    /* %e – zapis naukowy */
    printf("Pi w zapisie naukowym: %e\n", pi);

    /* %g – automatyczny wybór formy */
    printf("Pi automatycznie: %g\n", pi);

    /* %p – adres zmiennej (wskaźnik) */
    printf("Adres zmiennej ilosc: %p\n", (void *)&ilosc);

    /* %% – dosłowny znak procenta */
    printf("Podatek: 23%%\n");

    /* fprintf – drukuje do strumienia stderr (standardowe wyjście błędów) */
    fprintf(stderr, "To jest komunikat bledu\n");

    /* fprintf – drukuje do stdout (identycznie jak printf) */
    fprintf(stdout, "To jest normalny komunikat\n");

    /* Złożony przykład – tabela */
    printf("\n--- Tabela produktow ---\n");
    printf("%-15s %8s %10s\n", "Produkt", "Ilosc", "Cena");
    printf("%-15s %8d %10.2f\n", "Jabłka",    10, 2.49);
    printf("%-15s %8d %10.2f\n", "Banany",    5,  3.99);
    printf("%-15s %8d %10.2f\n", "Pomarancze", 7, 4.20);

    return EXIT_SUCCESS; /* poprawne zakończenie programu */
}
```

### Przykładowe wyjście

```

Cena: 12.99 PLN
Zakupiono   5 sztuki po 12.99 PLN.
Numer: 007
|witaj     |
|     witaj|
Litera: A
255 szesnastkowo: ff
255 szesnastkowo z prefiksem: 0xff
255 osemkowo: 377
Pi w zapisie naukowym: 3.141593e+00
Pi automatycznie: 3.14159
Adres zmiennej ilosc: 0x7ffd... (wartość zależy od systemu)
Podatek: 23%
To jest komunikat bledu
To jest normalny komunikat

--- Tabela produktow ---
Produkt            Ilosc       Cena
Jabłka                10       2.49
Banany                 5       3.99
Pomarancze             7       4.20
```

### Typowe błędy i jak ich unikać

**Błąd 1: Niezgodność specyfikatora z typem argumentu**
```c
/* ŹLE – long long wymaga %lld, nie %d */
long long duza = 9999999999LL;
printf("%d\n", duza);   /* niezdefiniowane zachowanie! */

/* DOBRZE */
printf("%lld\n", duza);
```

**Błąd 2: Brak & przy scanf (analogiczna funkcja wczytująca)**
```c
int x;
scanf("%d", x);   /* ŹLE – x zamiast &x, zapis w losowe miejsce pamięci */
scanf("%d", &x);  /* DOBRZE */
```

**Błąd 3: Drukowanie wskaźnika bez rzutowania na void***
```c
int *p = &x;
printf("%p\n", p);        /* ostrzeżenie kompilatora */
printf("%p\n", (void*)p); /* DOBRZE */
```

**Błąd 4: Zapomnienie o \n – brak opróżnienia bufora**
```c
printf("Wczytaj liczbe: "); /* bez \n może nie pojawić się przed scanf */
/* Lepiej: */
printf("Wczytaj liczbe: ");
fflush(stdout); /* wymusza opróżnienie bufora */
```

## 2. Zmienna liczba argumentów (variadic functions)

### Teoria

Funkcje ze zmienną liczbą argumentów (ang. *variadic functions*) pozwalają przyjąć dowolną liczbę parametrów – tak jak robi to `printf`. Mechanizm ten jest dostępny przez nagłówek `<stdarg.h>`.

Aby użyć tego mechanizmu, potrzebujemy:

- **`va_list`** – typ zmiennej przechowującej stan listy argumentów (wewnętrznie to wskaźnik wędrujący po stosie)
- **`va_start(va, ostatni_nazwany)`** – inicjuje listę; musimy podać ostatni nazwany parametr funkcji, żeby kompilator wiedział od którego miejsca na stosie zaczynają się zmienne argumenty
- **`va_arg(va, typ)`** – pobiera kolejny argument z listy; musimy podać dokładny typ, bo inaczej wskaźnik przesunie się o złą liczbę bajtów
- **`va_end(va)`** – kończy pracę z listą, zwalnia zasoby; **zawsze** należy to wywołać
- **`va_copy(dst, src)`** – kopiuje stan listy, przydatne gdy chcemy przejść listę dwukrotnie

**Ważne ograniczenie:** nie ma mechanizmu, który automatycznie powie ile argumentów przekazano. Programista musi sam zakodować tę informację – albo jako pierwszy parametr (liczba argumentów), albo jako specjalną wartość sentinel kończącą listę (np. NULL lub -1).

Zmienne argumenty podlegają *domyślnym promocjom*:
- `char` i `short` są promowane do `int`
- `float` jest promowany do `double`

Dlatego w `va_arg` należy pisać `va_arg(args, double)` a nie `va_arg(args, float)`.

### Kompletny działający kod

```c
#include <stdio.h>    /* printf */
#include <stdarg.h>   /* va_list, va_start, va_arg, va_end, va_copy */
#include <stdlib.h>   /* EXIT_SUCCESS */

/* Funkcja sumująca n liczb całkowitych */
/* n – ile argumentów zostanie przekazanych */
int suma_int(int n, ...) {
    va_list args;          /* deklaracja listy argumentów */
    va_start(args, n);     /* inicjacja: argumenty zaczynają się po 'n' */

    int wynik = 0;         /* akumulator sumy */

    for (int i = 0; i < n; i++) {          /* iterujemy dokładnie n razy */
        int liczba = va_arg(args, int);    /* pobieramy kolejny argument jako int */
        wynik += liczba;                   /* dodajemy do sumy */
    }

    va_end(args);          /* obowiązkowe zamknięcie listy */
    return wynik;
}

/* Funkcja sumująca liczby double; n to liczba argumentów */
double suma_double(int n, ...) {
    va_list args;           /* lista argumentów */
    va_start(args, n);      /* inicjacja po parametrze 'n' */

    double wynik = 0.0;     /* akumulator */

    for (int i = 0; i < n; i++) {
        /* UWAGA: float jest promowany do double, więc używamy double */
        double liczba = va_arg(args, double);
        wynik += liczba;
    }

    va_end(args);
    return wynik;
}

/* Funkcja drukująca n łańcuchów znaków oddzielonych separatorem */
void drukuj_napisy(const char *separator, int n, ...) {
    va_list args;                /* lista argumentów */
    va_start(args, n);           /* inicjacja po 'n' */

    for (int i = 0; i < n; i++) {
        const char *s = va_arg(args, const char *); /* pobieramy wskaźnik na napis */
        printf("%s", s);                            /* drukujemy napis */
        if (i < n - 1) printf("%s", separator);    /* separator między napisami */
    }
    printf("\n");

    va_end(args);
}

/* Demonstracja va_copy – przechodzenie listy dwukrotnie */
void drukuj_i_sumuj(int n, ...) {
    va_list args;          /* oryginalna lista */
    va_list kopia;         /* kopia listy */

    va_start(args, n);
    va_copy(kopia, args);  /* kopiujemy stan listy – teraz możemy użyć obu */

    printf("Elementy: ");
    for (int i = 0; i < n; i++) {
        int x = va_arg(args, int);  /* pobieramy z oryginału */
        printf("%d ", x);
    }
    printf("\n");

    int suma = 0;
    for (int i = 0; i < n; i++) {
        suma += va_arg(kopia, int); /* pobieramy z kopii (lista od początku) */
    }
    printf("Suma: %d\n", suma);

    va_end(args);   /* zamykamy oryginał */
    va_end(kopia);  /* zamykamy kopię */
}

int main(void) {

    /* Suma 4 liczb całkowitych: 10+20+30+40 */
    int s1 = suma_int(4, 10, 20, 30, 40);
    printf("Suma(4, 10, 20, 30, 40) = %d\n", s1);

    /* Suma 3 liczb: 1+2+3 */
    int s2 = suma_int(3, 1, 2, 3);
    printf("Suma(3, 1, 2, 3) = %d\n", s2);

    /* Suma liczb zmiennoprzecinkowych */
    double sd = suma_double(3, 1.5, 2.5, 3.0);
    printf("Suma double(3, 1.5, 2.5, 3.0) = %.2f\n", sd);

    /* Drukowanie napisów z separatorem */
    drukuj_napisy(", ", 4, "ala", "ma", "kota", "i psa");
    drukuj_napisy(" - ", 3, "jeden", "dwa", "trzy");

    /* Demonstracja va_copy */
    drukuj_i_sumuj(5, 10, 20, 30, 40, 50);

    return EXIT_SUCCESS;
}
```

### Przykładowe wyjście

```
Suma(4, 10, 20, 30, 40) = 100
Suma(3, 1, 2, 3) = 6
Suma double(3, 1.5, 2.5, 3.0) = 7.00
ala, ma, kota, i psa
jeden - dwa - trzy
Elementy: 10 20 30 40 50
Suma: 150
```

### Typowe błędy i jak ich unikać

**Błąd 1: Użycie float zamiast double w va_arg**
```c
/* ŹLE – float jest promowany do double na stosie! */
float f = va_arg(args, float);   /* niezdefiniowane zachowanie */

/* DOBRZE */
double f = va_arg(args, double); /* zawsze double dla liczb zmiennoprzecinkowych */
```

**Błąd 2: Zapomnienie va_end**
```c
va_start(args, n);
/* ... przetwarzanie ... */
/* zapomniałem va_end(args) – wyciek zasobów lub crash na niektórych platformach */
```

**Błąd 3: Pobranie złej liczby argumentów**
```c
/* Jeśli powiem suma_int(3, 10, 20) ale spróbuję pobrać 3 argumenty: */
/* trzecia iteracja pobierze śmieciową wartość ze stosu */
```

**Błąd 4: char i short w va_arg**
```c
/* ŹLE */
char c = va_arg(args, char);   /* char jest promowany do int */

/* DOBRZE */
char c = (char)va_arg(args, int);  /* pobierz jako int, potem zrzutuj */
```

## 3. Funkcja main – warianty

### Teoria

Funkcja `main` jest punktem wejścia każdego programu w C. Standard C definiuje dwa dopuszczalne podpisy:

**Wariant 1: `int main(void)`**
Używany gdy program nie potrzebuje argumentów linii poleceń. Słowo `void` w nawiasach jest tu jawnym oznaczeniem braku parametrów (w C, puste nawiasy `()` oznaczają nieznaną liczbę argumentów – to różni się od C++).

**Wariant 2: `int main(int argc, char *argv[])`**
Używany gdy program korzysta z argumentów podanych przy uruchomieniu. `argc` (*argument count*) to liczba argumentów, `argv` (*argument vector*) to tablica wskaźników na łańcuchy znaków.

Struktura `argv`:
- `argv[0]` – nazwa programu (tak jak go wywołano, np. `"./moj_program"`)
- `argv[1]` – pierwszy argument użytkownika
- `argv[2]` – drugi argument użytkownika
- `argv[argc]` – gwarantowanie `NULL` (wartownik końca tablicy)

Wartość zwracana przez `main`:
- `0` lub `EXIT_SUCCESS` – program zakończył się pomyślnie
- `1` lub `EXIT_FAILURE` – program zakończył się błędem
- Dowolna inna wartość – interpretacja zależy od systemu operacyjnego

### Kompletny działający kod

```c
#include <stdio.h>    /* printf, fprintf */
#include <stdlib.h>   /* EXIT_SUCCESS, EXIT_FAILURE, strtol */
#include <errno.h>    /* errno – kod błędu systemowego */

/* Program sumuje liczby całkowite podane jako argumenty wiersza poleceń */
/* Uruchomienie: ./program 12 25 78 */

int main(int argc, char *argv[]) {

    /* argc zawiera CAŁKOWITĄ liczbę argumentów, włącznie z nazwą programu */
    printf("Nazwa programu: %s\n", argv[0]);       /* argv[0] = nazwa programu */
    printf("Liczba argumentow (bez nazwy): %d\n", argc - 1);

    /* Sprawdzamy czy podano jakiekolwiek argumenty */
    if (argc < 2) {
        /* fprintf do stderr – błędy piszemy na stderr, nie stdout */
        fprintf(stderr, "Uzycie: %s liczba1 liczba2 ...\n", argv[0]);
        return EXIT_FAILURE; /* zakończ z kodem błędu */
    }

    /* Drukujemy wszystkie argumenty */
    printf("\nWszystkie argumenty:\n");
    for (int i = 1; i < argc; i++) {        /* zaczynamy od 1, bo 0 to nazwa */
        printf("  argv[%d] = \"%s\"\n", i, argv[i]);
    }

    /* Sumujemy tylko argumenty, które są poprawnymi liczbami */
    int suma = 0;
    printf("\nParsowanie:\n");

    for (int i = 1; i < argc; i++) {
        char *endptr;    /* wskaźnik na pierwszy znak, którego nie udało się sparsować */
        errno = 0;       /* zerujemy errno przed wywołaniem strtol */

        /* strtol – konwersja stringa na long; bezpieczniejsza niż atoi */
        /* argv[i] – string do konwersji */
        /* &endptr – tutaj strtol zapisze gdzie skończyła parsowanie */
        /* 10 – podstawa systemu liczbowego (dziesiętny) */
        long val = strtol(argv[i], &endptr, 10);

        /* Sprawdzamy poprawność konwersji: */
        /* errno == 0 – brak błędu przepełnienia */
        /* *endptr == '\0' – endptr wskazuje koniec stringa, czyli cały string był liczbą */
        if (errno == 0 && *endptr == '\0') {
            printf("  \"%s\" -> %ld (OK)\n", argv[i], val);
            suma += (int)val;   /* rzutujemy long na int */
        } else {
            printf("  \"%s\" -> blad konwersji, pomijam\n", argv[i]);
        }
    }

    printf("\nWynik sumy: %d\n", suma);

    /* Demonstracja argv[argc] == NULL */
    printf("\nargv[argc] = %s\n", argv[argc] == NULL ? "NULL (poprawnie)" : "nie-NULL");

    return EXIT_SUCCESS; /* sukces */
}
```

### Przykładowe wyjście

Dla wywołania: `./program 12 pomylka 25 78`

```

Liczba argumentow (bez nazwy): 4

Wszystkie argumenty:
  argv[1] = "12"
  argv[2] = "pomylka"
  argv[3] = "25"
  argv[4] = "78"

Parsowanie:
  "12" -> 12 (OK)
  "pomylka" -> blad konwersji, pomijam
  "25" -> 25 (OK)
  "78" -> 78 (OK)

Wynik sumy: 115

argv[argc] = NULL (poprawnie)
```

Dla wywołania bez argumentów: `./program`
```

Liczba argumentow (bez nazwy): 0
Uzycie: ./program liczba1 liczba2 ...
```

### Typowe błędy i jak ich unikać

**Błąd 1: Używanie atoi zamiast strtol**
```c
/* ŹLE – atoi nie informuje o błędzie; "abc" i "0" dają to samo: 0 */
int n = atoi(argv[1]);

/* DOBRZE – strtol informuje o błędzie przez endptr i errno */
char *end;
errno = 0;
long n = strtol(argv[1], &end, 10);
if (errno != 0 || *end != '\0') { /* błąd */ }
```

**Błąd 2: Dostęp do argv bez sprawdzenia argc**
```c
/* ŹLE – jeśli użytkownik nie podał argumentu, argv[1] == NULL */
printf("%s\n", argv[1]);   /* crash gdy argc == 1 */

/* DOBRZE */
if (argc > 1) printf("%s\n", argv[1]);
```

**Błąd 3: Modyfikacja argv[0]**
```c
argv[0][0] = 'X'; /* technicznie niezdefiniowane zachowanie – nie rób tego */
```

## 4. Makra preprocesora vs funkcje inline

### Teoria

**Makra (`#define`)** działają na etapie *preprocesora* – przed kompilacją. Preprocesor wykonuje dosłowne podstawienie tekstu. Makro `#define KWADRAT(x) ((x)*(x))` sprawia, że każde wystąpienie `KWADRAT(a+b)` w kodzie zostaje zamienione na `((a+b)*(a+b))` jeszcze przed tym, jak kompilator w ogóle zobaczy kod.

Zalety makr: działają dla każdego typu (brak kontroli typów może być zaletą w pewnych scenariuszach), zero narzutu wywołania funkcji.

Wady makr: brak kontroli typów, trudne debugowanie (debugger widzi rozwinięte makro), efekty uboczne przy wyrażeniach z inkrementacją.

**Funkcje inline** to sugestia dla kompilatora, aby wbudował ciało funkcji bezpośrednio w miejsce wywołania (unikając narzutu wywołania), ale przy zachowaniu wszystkich zalet normalnej funkcji: kontroli typów, poprawnego działania z inkrementacją, możliwości debugowania.

Słowo `inline` jest jedynie *życzeniem* – kompilator może je zignorować. Nowoczesne kompilatory i tak często inlinują funkcje bez tego słowa kluczowego, jeśli uzna to za korzystne.

**Kluczowa różnica z efektami ubocznymi:**
```
KWADRAT(x++)  =>  ((x++) * (x++))   // x inkrementowany DWUKROTNIE!
kwadrat(x++)  =>  x inkrementowany RAZ, do funkcji trafia wartość
```

### Kompletny działający kod

```c
#include <stdio.h>    /* printf */
#include <stdlib.h>   /* EXIT_SUCCESS */

/* Makro obliczające kwadrat – dosłowne podstawienie tekstu */
/* Zewnętrzne nawiasy chronią przed: p = 2 * KWADRAT(x) -> 2 * x * x (błąd priorytetu) */
/* Wewnętrzne nawiasy chronią przed: KWADRAT(a+b) -> a+b*a+b (błąd priorytetu) */
#define KWADRAT_MAKRO(x) ((x) * (x))

/* Makro MAX – klasyczne makro z efektem ubocznym */
#define MAX_MAKRO(a, b) ((a) > (b) ? (a) : (b))

/* Funkcja inline obliczająca kwadrat – bezpieczna wersja */
/* inline = sugestia dla kompilatora, żeby wbudował kod zamiast wywołania */
static inline int kwadrat_int(int x) {
    return x * x;   /* x jest obliczane dokładnie raz */
}

/* Funkcja inline dla double */
static inline double kwadrat_double(double x) {
    return x * x;
}

/* Funkcja inline MAX */
static inline int max_int(int a, int b) {
    return a > b ? a : b;
}

int main(void) {

    int bok = 5;

    /* --- Demonstracja makra --- */
    int p1 = KWADRAT_MAKRO(bok);       /* -> ((bok) * (bok)) */
    printf("KWADRAT_MAKRO(%d) = %d\n", bok, p1);

    int d = 3;
    int p2 = 2 * KWADRAT_MAKRO(d + 1); /* -> 2 * ((d+1) * (d+1)) = 2 * 16 = 32 */
    printf("2 * KWADRAT_MAKRO(%d+1) = %d\n", d, p2);

    /* --- Niebezpieczny efekt uboczny z makrem --- */
    int a = 4;
    int wynik_makro = KWADRAT_MAKRO(a++);
    /* Rozwinięcie: ((a++) * (a++))
       Wartość: 4 * 5 = 20 (ale może być też 4*4 lub UB – zależy od kompilatora!)
       'a' po operacji: 6 (bo a++ wykonane dwukrotnie) */
    printf("\nEfekt uboczny z makrem:\n");
    printf("  a przed: 4\n");
    printf("  KWADRAT_MAKRO(a++) = %d (niezdefiniowane zachowanie!)\n", wynik_makro);
    printf("  a po: %d (powinno byc 5, ale jest %d!)\n", a, a);

    /* --- Bezpieczna funkcja inline --- */
    int b = 4;
    int wynik_inline = kwadrat_int(b++);
    /* b++ jest obliczane RAZ przed wywołaniem funkcji
       do funkcji trafia wartość 4, wynik = 16
       b po operacji = 5 */
    printf("\nFunkcja inline:\n");
    printf("  b przed: 4\n");
    printf("  kwadrat_int(b++) = %d\n", wynik_inline);
    printf("  b po: %d (poprawnie 5)\n", b);

    /* --- Makro działa dla różnych typów --- */
    double x = 2.5;
    double pd = KWADRAT_MAKRO(x);     /* działa też dla double */
    printf("\nKWADRAT_MAKRO(2.5) = %.2f\n", pd);

    /* --- Funkcja inline wymaga odpowiedniego typu --- */
    double xd = 2.5;
    double pd2 = kwadrat_double(xd);  /* używamy wersji double */
    printf("kwadrat_double(2.5) = %.2f\n", pd2);

    /* --- MAX --- */
    printf("\nMAX_MAKRO(3, 7) = %d\n", MAX_MAKRO(3, 7));
    printf("max_int(3, 7) = %d\n", max_int(3, 7));

    /* Niebezpieczny MAX z makrem */
    int c = 3, e = 7;
    int m = MAX_MAKRO(c++, e++);
    /* Rozwinięcie: ((c++) > (e++) ? (c++) : (e++))
       e > c, więc wynik = e++ (czyli e jest inkrementowane DWUKROTNIE!) */
    printf("\nMAX_MAKRO(c++=%d, e++=%d) = %d, c=%d, e=%d\n", 3, 7, m, c, e);

    return EXIT_SUCCESS;
}
```

## 5. Wskaźnik do funkcji

### Teoria

W C funkcja to blok kodu umieszczony w pamięci pod konkretnym adresem. Wskaźnik do funkcji to zmienna, która przechowuje ten adres. Dzięki temu możemy:
- przekazywać funkcje jako argumenty do innych funkcji (np. komparator do `qsort`)
- przechowywać zestawy funkcji w tablicach (np. tablica akcji w grze)
- implementować proste "polimorfizm" w C

Składnia deklaracji wskaźnika do funkcji:
```c
typ_zwracany (*nazwa_wskaznika)(typy_parametrow);
```

Przykłady:
```c
int (*p)(void);           /* wskaźnik do funkcji zwracającej int, bez parametrów */
void (*p)(int, double);   /* wskaźnik do funkcji void z int i double */
int (*tab[10])(int);      /* tablica 10 wskaźników do funkcji int(int) */
```

Gwiazdka musi być w nawiasie: `int *p(int)` to deklaracja funkcji zwracającej `int*`, nie wskaźnik!

Wywołanie przez wskaźnik: `(*wsk)(argumenty)` lub skrótowo `wsk(argumenty)`.

### Kompletny działający kod

```c
#include <stdio.h>    /* printf */
#include <stdlib.h>   /* EXIT_SUCCESS, qsort */
#include <string.h>   /* strcmp */

/* Kilka prostych funkcji matematycznych */
int dodaj(int a, int b) { return a + b; }         /* dodawanie */
int odejmij(int a, int b) { return a - b; }       /* odejmowanie */
int pomnoz(int a, int b) { return a * b; }        /* mnożenie */

/* Funkcja przyjmująca wskaźnik do funkcji jako argument */
/* Wzorzec: typ_wyniku (*nazwa)(typy_parametrow) */
void pokaz_wynik(int a, int b, int (*operacja)(int, int), const char *nazwa) {
    int wynik = operacja(a, b);   /* wywołanie przez wskaźnik */
    printf("%s(%d, %d) = %d\n", nazwa, a, b, wynik);
}

/* Funkcje porównujące – wzorzec wymagany przez qsort */
/* qsort wymaga: int komparator(const void*, const void*) */
int porownaj_rosnaco(const void *a, const void *b) {
    /* rzutujemy void* na int*, wyłuskujemy wartości */
    int x = *(const int *)a;   /* wartość pierwszego elementu */
    int y = *(const int *)b;   /* wartość drugiego elementu */
    return x - y;              /* ujemny jeśli a<b, 0 jeśli równe, dodatni jeśli a>b */
}

int porownaj_malejaco(const void *a, const void *b) {
    int x = *(const int *)a;
    int y = *(const int *)b;
    return y - x;              /* odwrócona kolejność */
}

/* Tablica wskaźników do funkcji – przykład "menu akcji" */
void akcja_nic(void)    { printf("Nie robimy nic\n"); }
void akcja_hop(void)    { printf("Skaczemy!\n"); }
void akcja_bieg(void)   { printf("Biegniemy!\n"); }
void akcja_stop(void)   { printf("Zatrzymujemy sie!\n"); }

int main(void) {

    /* --- Podstawowa deklaracja i użycie wskaźnika --- */
    int (*p)(int, int);    /* deklaracja wskaźnika do funkcji int(int,int) */

    p = dodaj;             /* przypisanie adresu funkcji (bez nawiasów!) */
    printf("Przez wskaznik: p(3, 4) = %d\n", p(3, 4));   /* wywołanie skrótowe */
    printf("Przez wskaznik: (*p)(3, 4) = %d\n", (*p)(3, 4)); /* wywołanie klasyczne */

    p = odejmij;           /* zmiana wskazywanej funkcji */
    printf("Po zmianie: p(10, 3) = %d\n", p(10, 3));

    /* --- Wskaźnik jako argument funkcji --- */
    printf("\nPrzekazywanie wskaznika jako argumentu:\n");
    pokaz_wynik(10, 3, dodaj,    "dodaj");
    pokaz_wynik(10, 3, odejmij,  "odejmij");
    pokaz_wynik(10, 3, pomnoz,   "pomnoz");

    /* --- Tablica wskaźników do funkcji --- */
    printf("\nTablica wskaznikow do funkcji:\n");

    /* Deklaracja tablicy 4 wskaźników do funkcji void(void) */
    void (*akcje[4])(void);

    akcje[0] = akcja_nic;    /* wypełniamy tablicę adresami funkcji */
    akcje[1] = akcja_hop;
    akcje[2] = akcja_bieg;
    akcje[3] = akcja_stop;

    /* Wywołujemy funkcje przez tablicę wskaźników */
    for (int i = 0; i < 4; i++) {
        printf("  akcje[%d]: ", i);
        akcje[i]();           /* wywołanie i-tej akcji */
    }

    /* Możemy też tworzyć "mapping" – inna tablica wskaźników do tych samych funkcji */
    void (*sekwencja[4])(void);
    sekwencja[0] = akcje[2];   /* bieg */
    sekwencja[1] = akcje[1];   /* hop */
    sekwencja[2] = akcje[3];   /* stop */
    sekwencja[3] = akcje[0];   /* nic */

    printf("\nSekwencja przez mapowanie:\n");
    for (int i = 0; i < 4; i++) {
        printf("  sekwencja[%d]: ", i);
        sekwencja[i]();
    }

    /* --- qsort z wskaźnikiem do komparatora --- */
    int tablica[] = {5, 2, 8, 1, 9, 3, 7, 4, 6};
    int n = sizeof(tablica) / sizeof(tablica[0]); /* liczba elementów */

    /* qsort(ptr, n_elementow, rozmiar_elementu, komparator) */
    qsort(tablica, n, sizeof(int), porownaj_rosnaco);

    printf("\nPo sortowaniu rosnacym: ");
    for (int i = 0; i < n; i++) printf("%d ", tablica[i]);
    printf("\n");

    qsort(tablica, n, sizeof(int), porownaj_malejaco);

    printf("Po sortowaniu malejacym: ");
    for (int i = 0; i < n; i++) printf("%d ", tablica[i]);
    printf("\n");

    /* --- typedef dla czytelności --- */
    typedef int (*Operacja)(int, int);  /* nowy typ: wskaźnik do funkcji int(int,int) */

    Operacja ops[] = {dodaj, odejmij, pomnoz};    /* tablica z typedef */
    const char *nazwy[] = {"dodaj", "odejmij", "pomnoz"};

    printf("\nWszystkie operacje na (6, 2):\n");
    for (int i = 0; i < 3; i++) {
        printf("  %s(6, 2) = %d\n", nazwy[i], ops[i](6, 2));
    }

    return EXIT_SUCCESS;
}
```

**Błąd 1: Pomylenie wskaźnika do funkcji z funkcją zwracającą wskaźnik**
```c

```


```c
double fun(double x) { return x; }
```

## 6. Wskaźnik do struktury

### Teoria

Wskaźnik do struktury to zmienna przechowująca adres struktury w pamięci. Jest to bardzo powszechny wzorzec w C, używany m.in. do:
- unikania kopiowania dużych struktur przy przekazywaniu do funkcji
- tworzenia list połączonych, drzew i innych dynamicznych struktur danych
- modyfikowania struktury wewnątrz funkcji

Dostęp do pól przez wskaźnik można realizować na dwa sposoby:
- `(*wsk).pole` – wyłuskaj wskaźnik, potem wybierz pole (nawiasy obowiązkowe ze względu na priorytety)
- `wsk->pole` – operator strzałki, cukier składniowy dla `(*wsk).pole`; czytelniejszy i używany znacznie częściej

### Kompletny działający kod

```c
#include <stdio.h>     /* printf, puts */
#include <string.h>    /* strcpy */
#include <stdlib.h>    /* EXIT_SUCCESS */

/* Definicja struktury opisującej miejsce geograficzne */
typedef struct {
    char nazwa[40];   /* nazwa miejsca */
    float dl_geo;     /* długość geograficzna */
    float sz_geo;     /* szerokość geograficzna */
    int ocena;        /* ocena od 1 do 10 */
} t_miejsce;

/* Funkcja przyjmująca wskaźnik do struktury – może ją modyfikować */
void wypisz_miejsce(const t_miejsce *m) {
    /* const t_miejsce* – wskaźnik na stałą strukturę, nie modyfikujemy */
    printf("Miejsce: %s\n", m->nazwa);                           /* operator -> */
    printf("  Wspolrzedne: %.4f, %.4f\n", m->dl_geo, m->sz_geo);
    printf("  Ocena: %d/10\n", m->ocena);
}

/* Funkcja modyfikująca strukturę przez wskaźnik */
void ustaw_ocene(t_miejsce *m, int nowa_ocena) {
    /* Bez const – możemy modyfikować strukturę */
    m->ocena = nowa_ocena;   /* modyfikacja przez wskaźnik */
}

/* Funkcja drukująca nazwę – oba sposoby dostępu */
void pokaz_dostep(t_miejsce *wsk) {
    puts((*wsk).nazwa);    /* sposób 1: wyłuskaj, potem wybierz pole */
    puts(wsk->nazwa);      /* sposób 2: operator strzałki – czytelniejszy */
}

int main(void) {

    /* Inicjalizacja struktury z użyciem designated initializers */
    t_miejsce Kalisz = {
        .nazwa = "Kalisz",
        .dl_geo = 18.0910f,    /* długość geograficzna */
        .sz_geo = 51.7611f,    /* szerokość geograficzna */
        .ocena = 5
    };

    /* Deklaracja wskaźnika i przypisanie adresu struktury */
    t_miejsce *wsk = &Kalisz;   /* wsk wskazuje na strukturę Kalisz */

    /* Dostęp przez wskaźnik – oba sposoby są równoważne */
    printf("Przez (*wsk).nazwa: %s\n", (*wsk).nazwa);
    printf("Przez wsk->nazwa:   %s\n", wsk->nazwa);

    /* Modyfikacja przez wskaźnik */
    wsk->ocena = 8;              /* zmienia Kalisz.ocena na 8 */
    printf("Nowa ocena: %d\n", Kalisz.ocena);  /* Kalisz.ocena == 8 */

    /* Adres pola struktury – potrzebny np. do scanf */
    printf("\nAdres pola ocena: %p\n", (void *)&(wsk->ocena));
    /* &(wsk->ocena) == &((*wsk).ocena) == &(Kalisz.ocena) – to samo! */

    /* Wywołanie funkcji ze wskaźnikiem */
    printf("\nWypisanie przez funkcje:\n");
    wypisz_miejsce(wsk);

    ustaw_ocene(wsk, 9);         /* ustawia ocenę przez wskaźnik */
    printf("Ocena po ustaw_ocene: %d\n", Kalisz.ocena);

    /* Tablica struktur i wskaźnik na element tablicy */
    t_miejsce miasta[] = {
        {"Krakow", 19.9450f, 50.0647f, 10},
        {"Warszawa", 21.0122f, 52.2297f, 8},
        {"Gdansk", 18.6464f, 54.3520f, 9}
    };

    t_miejsce *p = miasta;    /* p wskazuje na pierwszy element tablicy */

    printf("\nIteracja po tablicy struktur:\n");
    for (int i = 0; i < 3; i++) {
        printf("  p->nazwa = %s, ocena = %d\n", p->nazwa, p->ocena);
        p++;   /* przesuwamy wskaźnik na następny element tablicy */
    }

    /* Wskaźnik NULL – bezpieczne sprawdzanie */
    t_miejsce *pusty = NULL;
    if (pusty != NULL) {
        pusty->ocena = 5;   /* ten kod nie zostanie wykonany */
    } else {
        printf("\nWskaznik NULL – nie mozna dereferencjonowac!\n");
    }

    return EXIT_SUCCESS;
}
```

### Przykładowe wyjście

```

Przez wsk->nazwa:   Kalisz
Nowa ocena: 8

Adres pola ocena: 0x7ffd... (zależy od systemu)

Wypisanie przez funkcje:
Miejsce: Kalisz
  Wspolrzedne: 18.0910, 51.7611
  Ocena: 8/10
Ocena po ustaw_ocene: 9

Iteracja po tablicy struktur:
  p->nazwa = Krakow, ocena = 10
  p->nazwa = Warszawa, ocena = 8
  p->nazwa = Gdansk, ocena = 9

Wskaznik NULL – nie mozna dereferencjonowac!
```

### Typowe błędy i jak ich unikać

**Błąd 1: Dereferencja wskaźnika NULL**
```c
t_miejsce *wsk = NULL;
wsk->ocena = 5;   /* crash – segmentation fault */

/* Zawsze sprawdzaj NULL przed użyciem */
if (wsk != NULL) wsk->ocena = 5;
```

**Błąd 2: Priorytety – brak nawiasów przy (*wsk)**

**Błąd 3: Zwracanie wskaźnika do lokalnej zmiennej**

## 7. Struktury zagnieżdżone i anonimowe

### Teoria

**Struktury zagnieżdżone** to struktury, które zawierają pola będące innymi strukturami. Pozwala to modelować złożone obiekty ze świata rzeczywistego (osoba ma adres, adres ma miasto itd.). Inicjalizacja pól zagnieżdżonych odbywa się przez *designated initializers* z kropką: `.pole.podpole = wartość`.

**Struktury anonimowe** (C11+) to struktury zdefiniowane wewnątrz innej struktury *bez nazwy*. Ich pola stają się bezpośrednio dostępne w strukturze zewnętrznej – bez konieczności podawania nazwy struktury pośredniej. Jest to wygodne gdy kilka pól logicznie się grupuje, ale nie chcemy tworzyć osobnego nazwanego podtypu.

```c
struct Zewnetrzna {
    int x;
    struct {        /* anonimowa struktura – brak nazwy */
        int a;
        int b;
    };              /* brak nazwy pola! */
};

struct Zewnetrzna e;
e.x = 1;
e.a = 2;   /* bezpośredni dostęp – nie e.cos.a */
e.b = 3;
```

### Kompletny działający kod

```c
#include <stdio.h>     /* printf */
#include <stdlib.h>    /* EXIT_SUCCESS */

/* Pomocnicza struktura przewodnika */
struct t_przewodnik {
    char imie[20];      /* imię przewodnika */
    char nazwisko[20];  /* nazwisko przewodnika */
    char kontakt[60];   /* dane kontaktowe */
};

/* Struktura miejsca z zagnieżdżonymi strukturami */
struct t_miejsce {
    char nazwa[40];                           /* nazwa miejsca */
    float dl_geo;                             /* długość geograficzna */
    float sz_geo;                             /* szerokość geograficzna */
    int ocena;                                /* ocena miejsca */
    struct t_przewodnik nasz_przewodnik;      /* JEDNA zagnieżdżona struktura */
    struct t_przewodnik przewodnicy[3];       /* TABLICA zagnieżdżonych struktur */
};

/* Struktura z anonimową strukturą wewnątrz */
struct Osoba {
    char imie[20];
    char nazwisko[30];

    struct t_data {      /* nazwana wewnętrzna struktura */
        int dzien;
        int miesiac;
        int rok;
    };                   /* ANONIMOWA – brak nazwy pola, pola są widoczne bezpośrednio */

    struct t_adres {
        char miasto[20];
        int ile_lat;
    } adres;             /* NIE-anonimowa – pole nazywa się 'adres' */
};

/* Funkcja wypisująca zagnieżdżoną strukturę */
void wypisz_miejsce(const struct t_miejsce *m) {
    printf("Miejsce: %s (%.2f, %.2f), ocena: %d\n",
           m->nazwa, m->dl_geo, m->sz_geo, m->ocena);

    /* Dostęp do zagnieżdżonej struktury: m->nasz_przewodnik.imie */
    printf("  Glowny przewodnik: %s %s\n",
           m->nasz_przewodnik.imie,
           m->nasz_przewodnik.nazwisko);

    /* Iteracja po tablicy zagnieżdżonych struktur */
    for (int i = 0; i < 3; i++) {
        if (m->przewodnicy[i].imie[0] != '\0') {  /* sprawdzamy czy pole jest niepuste */
            printf("  Przewodnik[%d]: %s %s\n",
                   i,
                   m->przewodnicy[i].imie,      /* dostęp: pole_tablicy.pole */
                   m->przewodnicy[i].nazwisko);
        }
    }
}

int main(void) {

    /* Inicjalizacja struktury zagnieżdżonej – designated initializers */
    struct t_miejsce elk = {
        .nazwa = "Elk",                          /* pole proste */
        .dl_geo = 22.3667f,
        .sz_geo = 53.8310f,
        .ocena = 10,
        .nasz_przewodnik.imie = "Ela",           /* zagnieżdżone przez kropkę */
        .nasz_przewodnik.nazwisko = "Kowalska",
        .nasz_przewodnik.kontakt = "ela@example.com",
        .przewodnicy[0].nazwisko = "Mars",       /* tablica zagnieżdżona */
        .przewodnicy[0].imie = "Jan",
        .przewodnicy[1].imie = "Anna",
        .przewodnicy[1].nazwisko = "Nowak"
        /* przewodnicy[2] będzie wyzerowany (C gwarantuje) */
    };

    wypisz_miejsce(&elk);

    /* --- Modyfikacja przez wskaźnik zagnieżdżonej struktury --- */
    struct t_miejsce *wsk = &elk;
    wsk->nasz_przewodnik.ocena_pracy = 0; /* gdybyśmy mieli takie pole */

    /* Poprawna modyfikacja istniejącego pola: */
    wsk->ocena = 9;
    printf("\nPo zmianie oceny na 9: %d\n", elk.ocena);

    /* Dostęp alternatywny – przez wskaźnik zagnieżdżonej struktury */
    struct t_przewodnik *p = &(wsk->nasz_przewodnik);
    printf("Imie glownego przewodnika przez wskaznik: %s\n", p->imie);

    /* --- Struktura z anonimową strukturą wewnątrz --- */
    struct Osoba jan = {
        .imie = "Jan",
        .nazwisko = "Kowalski",
        /* pola anonimowej struktury t_data dostępne bezpośrednio */
        .dzien = 13,
        .miesiac = 4,
        .rok = 2025,
        /* pole .adres – nie anonimowe, trzeba podać nazwę */
        .adres.miasto = "Oslo",
        .adres.ile_lat = 3
    };

    /* Dostęp do pól anonimowej struktury – BEZ pośredniego nazwy */
    printf("\nOsoba: %s %s\n", jan.imie, jan.nazwisko);
    printf("Data urodzenia: %02d.%02d.%04d\n",
           jan.dzien,      /* bezpośrednio! (nie jan.data.dzien) */
           jan.miesiac,
           jan.rok);

    /* Dostęp do pól NIE-anonimowej struktury – PRZEZ nazwę 'adres' */
    printf("Mieszka w: %s od %d lat\n",
           jan.adres.miasto,    /* jan.adres.miasto – przez nazwę pola */
           jan.adres.ile_lat);

    return EXIT_SUCCESS;
}
```

## 8. Struktury rekurencyjne

### Teoria

Struktura rekurencyjna (samoreferencja) to struktura, która zawiera wskaźnik do struktury tego samego typu. Nie może zawierać bezpośredniej kopii samej siebie (bo miałaby nieskończony rozmiar), ale wskaźnik ma stały rozmiar (4 lub 8 bajtów), więc jest to możliwe.

Struktury rekurencyjne są fundamentem dla:
- **list połączonych** (każdy węzeł wskazuje na następny)
- **drzew binarnych** (każdy węzeł wskazuje na lewe i prawe dziecko)
- **grafów** (węzeł wskazuje na listę sąsiadów)

Kluczowe: wewnątrz definicji `struct t_miejsce` możemy użyć `struct t_miejsce *` (wskaźnik), bo rozmiar wskaźnika jest znany. Nie możemy użyć `struct t_miejsce` (wartość), bo rozmiar całej struktury nie jest jeszcze znany.

### Kompletny działający kod

```c
#include <stdio.h>     /* printf */
#include <stdlib.h>    /* EXIT_SUCCESS, malloc, free */
#include <string.h>    /* strcpy */

/* Struktura rekurencyjna – węzeł listy połączonej */
struct t_miejsce {
    char nazwa[40];              /* dane węzła */
    float dl_geo;
    float sz_geo;
    int ocena;
    struct t_miejsce *nastepny;  /* WSKAŹNIK na następny węzeł (nie wartość!) */
};

/* Funkcja tworząca nowy węzeł */
struct t_miejsce* nowy_wezel(const char *nazwa, float dl, float sz, int ocena) {
    struct t_miejsce *w = malloc(sizeof(struct t_miejsce));  /* alokacja pamięci */
    if (w == NULL) return NULL;          /* sprawdzamy czy malloc się powiódł */

    /* Kopiujemy dane do nowego węzła */
    /* strcpy zamiast = bo nazwa to tablica, nie wskaźnik */
    /* strncpy byłoby bezpieczniejsze, ale dla uproszczenia używamy strcpy */
    strcpy(w->nazwa, nazwa);
    w->dl_geo = dl;
    w->sz_geo = sz;
    w->ocena = ocena;
    w->nastepny = NULL;   /* nowy węzeł nie ma następnika */

    return w;
}

/* Funkcja wypisująca całą listę */
void wypisz_liste(const struct t_miejsce *glowa) {
    const struct t_miejsce *biezacy = glowa;   /* zaczynamy od głowy */

    int i = 0;
    while (biezacy != NULL) {            /* idziemy dopóki nie trafimy na NULL */
        printf("%d: %s (ocena: %d)\n", i, biezacy->nazwa, biezacy->ocena);
        biezacy = biezacy->nastepny;     /* przesuwamy się na następny węzeł */
        i++;
    }
}

/* Funkcja zwalniająca całą listę */
void zwolnij_liste(struct t_miejsce *glowa) {
    struct t_miejsce *biezacy = glowa;
    while (biezacy != NULL) {
        struct t_miejsce *nastepny = biezacy->nastepny;  /* zapamiętujemy następnik */
        free(biezacy);                                    /* zwalniamy bieżący węzeł */
        biezacy = nastepny;                               /* przechodzimy do następnego */
    }
}

int main(void) {

    /* --- Przykład z wykładu: dwa węzły na stosie --- */
    struct t_miejsce Elk  = {"Elk",  22.3667f, 53.8310f, 8, NULL};
    struct t_miejsce Puck = {"Puck", 18.4028f, 54.7300f, 9, NULL};

    /* Łączymy: Elk wskazuje na Puck */
    Elk.nastepny = &Puck;

    printf("Prosta lista ze stosu:\n");
    printf("  %s -> %s -> NULL\n",
           Elk.nazwa,
           Elk.nastepny->nazwa);   /* Elk.nastepny to Puck */

    /* Dostęp do pola następnego węzła: dwa poziomy -> */
    printf("  Nastepnik Elk: %s (ocena: %d)\n",
           Elk.nastepny->nazwa,    /* nazwa Puck */
           Elk.nastepny->ocena);   /* ocena Puck */

    /* --- Dynamiczna lista połączona --- */
    printf("\nDynamiczna lista polaczona:\n");

    /* Tworzymy węzły dynamicznie */
    struct t_miejsce *glowa = nowy_wezel("Krakow",   19.9450f, 50.0647f, 10);
    struct t_miejsce *w2    = nowy_wezel("Warszawa", 21.0122f, 52.2297f, 8);
    struct t_miejsce *w3    = nowy_wezel("Gdansk",   18.6464f, 54.3520f, 9);

    /* Łączymy węzły: Krakow -> Warszawa -> Gdansk -> NULL */
    glowa->nastepny = w2;
    w2->nastepny = w3;
    /* w3->nastepny == NULL (ustawione w nowy_wezel) */

    wypisz_liste(glowa);   /* wypisujemy całą listę */

    /* Przejście listy i znalezienie maksymalnej oceny */
    int max_ocena = 0;
    const char *najlepsze = "";
    struct t_miejsce *p = glowa;

    while (p != NULL) {
        if (p->ocena > max_ocena) {
            max_ocena = p->ocena;
            najlepsze = p->nazwa;
        }
        p = p->nastepny;   /* następny węzeł */
    }
    printf("Najwyzsza ocena: %s (%d)\n", najlepsze, max_ocena);

    /* Zwalniamy całą listę – obowiązkowo! */
    zwolnij_liste(glowa);
    printf("Pamiec zwolniona.\n");

    return EXIT_SUCCESS;
}
```

### Przykładowe wyjście

```
Prosta lista ze stosu:
  Elk -> Puck -> NULL
  Nastepnik Elk: Puck (ocena: 9)

Dynamiczna lista polaczona:
0: Krakow (ocena: 10)
1: Warszawa (ocena: 8)
2: Gdansk (ocena: 9)
Najwyzsza ocena: Krakow (10)
Pamiec zwolniona.
```

### Typowe błędy i jak ich unikać

**Błąd 1: Niezwolnienie pamięci – wyciek**
```c
struct t_miejsce *w = malloc(sizeof(*w));
/* ... używamy ... */
/* zapomnieliśmy free(w) – wyciek pamięci */
```

**Błąd 2: Użycie zwolnionej pamięci (use-after-free)**
```c
free(w);
printf("%s\n", w->nazwa);  /* ŹLE – w jest już zwolnione */
w = NULL;                  /* po free ustaw NULL, żeby przypadkowo nie użyć */
```

**Błąd 3: Utrata głowy listy**
```c

    free(glowa);
    glowa = glowa->nastepny;  /* ŹLE – glowa już zwolniona! */
}
/* Zawsze zapamiętaj następnik PRZED free */
```

## 9. Degradacja tablicy (Array Decay)

### Teoria

**Array Decay** (degradacja tablicy) to automatyczna konwersja nazwy tablicy na wskaźnik do jej pierwszego elementu. Dzieje się to w niemal każdym kontekście poza:
- operatorem `sizeof`
- operatorem `&`
- inicjalizacją tablicy znaków łańcuchem (`char s[] = "hello"`)

Konsekwencja: gdy przekazujemy tablicę do funkcji, funkcja **nigdy nie otrzymuje kopii tablicy** – zawsze otrzymuje wskaźnik. Rozmiar tablicy jest tracony! Dlatego zazwyczaj przekazujemy rozmiar jako osobny parametr.

**Ważna zasada przy deklarowaniu parametrów:**
- `int arr[]` i `int *arr` jako parametr funkcji – **identyczne** (oba to wskaźnik)
- `int arr[10]` jako parametr – kompilator **ignoruje** liczbę 10 i traktuje jak wskaźnik. To jest często oznaczane jako błąd przez nowoczesne kompilatory (a przynajmniej generuje ostrzeżenie), bo 10 jest mylące.

**Tablice wielowymiarowe:** przy tablicach 2D, po degradacji otrzymujemy wskaźnik na wiersz (`int (*)[liczba_kolumn]`). Numer kolumn MUSI być znany. `int tab[10][20]` degraduje do `int (*)[20]`.

### Kompletny działający kod

```c
#include <stdio.h>    /* printf */
#include <stdlib.h>   /* EXIT_SUCCESS */

/* Poprawna deklaracja – 'ids' to wskaźnik (ile musi być przekazane osobno) */
void wypisz_gracze_v1(int ile, int *ids, int *wiek) {
    /* ids[] i *ids w parametrze to to samo – wskaźnik na int */
    for (int i = 0; i < ile; i++) {
        printf("  Gracz %d ma %d lat\n", ids[i], wiek[i]);
        /* ids[i] == *(ids + i) -- oba zapisy równoważne */
    }
}

/* Alternatywna wersja z arytmetyką wskaźnikową */
void wypisz_gracze_v2(int ile, int *ids, int *wiek) {
    for (int i = 0; i < ile; i++) {
        /* *(ids+i) == ids[i] */
        printf("  Gracz %d ma %d lat\n", *(ids + i), *(wiek + i));
    }
}

/* Tablica 2D – musimy podać rozmiar drugiego wymiaru */
/* char imie[][20] – poprawnie: ile wierszy nie wiadomo, ale kolumn = 20 */
void wypisz_z_imieniem(int ile, char imie[][20], int *wiek) {
    for (int i = 0; i < ile; i++) {
        printf("  Gracz '%s' ma %d lat\n", imie[i], wiek[i]);
        /* imie[i] to i-ty wiersz tablicy char[][20] */
    }
}

/* sizeof w funkcji vs poza nią */
void pokaz_sizeof(int arr[], int n) {
    /* WEWNĄTRZ funkcji: arr to wskaźnik, sizeof(arr) = sizeof(int*) = 4 lub 8 */
    printf("  sizeof(arr) wewnatrz funkcji = %zu (rozmiar wskaznika!)\n",
           sizeof(arr));
    printf("  Rozmiar przekazany jako n = %d\n", n);
}

int main(void) {

    int ids[]  = {1, 2, 3, 4, 5};        /* tablica id graczy */
    int wiek[] = {20, 22, 19, 25, 21};   /* tablica wieku graczy */
    int n = sizeof(ids) / sizeof(ids[0]); /* liczba elementów – tylko tutaj działa! */

    printf("sizeof(ids) na zewnatrz funkcji = %zu\n", sizeof(ids));  /* 20 bajtów */
    printf("Liczba elementow n = %d\n", n);                           /* 5 */

    /* Wywołanie – tablica 'degraduje' do wskaźnika na pierwszy element */
    printf("\nWersja 1 (indeksowanie):\n");
    wypisz_gracze_v1(n, ids, wiek);

    printf("\nWersja 2 (arytmetyka wskaznikow):\n");
    wypisz_gracze_v2(n, ids, wiek);

    /* sizeof wewnątrz funkcji */
    printf("\nDemonstracja sizeof:\n");
    pokaz_sizeof(ids, n);

    /* Tablica 2D z imionami */
    char imiona[4][20] = {
        "Marek",
        "Ania",
        "Piotr",
        "Kasia"
    };
    int wieki[] = {22, 19, 25, 21};

    printf("\nGracze z imionami:\n");
    wypisz_z_imieniem(4, imiona, wieki);

    /* Demonstracja degradacji */
    printf("\nAdres tablicy i adres pierwszego elementu:\n");
    printf("  ids       = %p (adres tablicy)\n", (void *)ids);
    printf("  &ids[0]   = %p (adres 1. elementu)\n", (void *)&ids[0]);
    printf("  &ids      = %p (adres calej tablicy – ten sam adres, inny typ!)\n",
           (void *)&ids);

    /* Arytmetyka na wskaźnikach */
    int *p = ids;   /* p wskazuje na ids[0] */
    printf("\nArytmetyka wskaznikow:\n");
    for (int i = 0; i < n; i++) {
        printf("  *(p + %d) = %d\n", i, *(p + i));
    }

    return EXIT_SUCCESS;
}
```

### Typowe błędy i jak ich unikać

**Błąd 1: Obliczanie rozmiaru tablicy w funkcji przez sizeof**

```c
    int n = sizeof(arr) / sizeof(arr[0]);  /* ŹLE – sizeof(arr) = sizeof(int*) */
```

```c
/* lub */
void fun(int arr[][20], int wiersze) { }
```

## 10. Funkcje kończące program – exit, noreturn

### Teoria

**`noreturn`** (od C11, wcześniej `_Noreturn`) to specyfikator informujący kompilator, że dana funkcja **nigdy nie powróci** do miejsca wywołania. Kompilator może dzięki temu lepiej optymalizować kod i ostrzegać gdy taka funkcja faktycznie zwróciłaby wartość.

Funkcje `noreturn` to m.in.: `exit()`, `abort()`, `quick_exit()`, `_Exit()`, a także własne funkcje kończące program.

**`exit(status)`** – czyste zakończenie programu:
1. Wywołuje wszystkie funkcje zarejestrowane przez `atexit()` (w odwrotnej kolejności rejestracji)
2. Opróżnia i zamyka wszystkie strumienie wejścia-wyjścia
3. Usuwa pliki tymczasowe (`tmpfile`)
4. Przekazuje `status` do systemu operacyjnego

**`atexit(funkcja)`** – rejestruje funkcję, która zostanie wywołana przy normalnym kończeniu programu. Przydatne do sprzątania zasobów.

**`abort()`** – natychmiastowe, "niegrzeczne" zakończenie. Nie wywołuje `atexit`. Wysyła sygnał `SIGABRT`. Używane przy nieodwracalnych błędach.

**`quick_exit(status)`** – podobne do `exit`, ale wywołuje tylko funkcje zarejestrowane przez `at_quick_exit()`, nie `atexit()`. Przydatne w programach wielowątkowych.

Kody zakończenia:
- `EXIT_SUCCESS` (= 0) – sukces, przenośne między systemami
- `EXIT_FAILURE` (= 1) – błąd, przenośne między systemami

### Kompletny działający kod

```c
#include <stdio.h>     /* printf, puts */
#include <stdlib.h>    /* exit, atexit, abort, EXIT_SUCCESS, EXIT_FAILURE */

/* Funkcja zarejestrowana w atexit – wywoływana przy exit() lub return z main */
/* Musi mieć sygnaturę: void funkcja(void) */
void sprzataj_baze_danych(void) {
    puts("Zamykanie polaczenia z baza danych...");
}

void zapisz_logi(void) {
    puts("Zapisywanie logow...");
}

void komunikat_konca(void) {
    puts("Program zakonczony prawidlowo.");
}

/* Funkcja noreturn – nigdy nie powraca do wywołującego */
/* Słowo noreturn musi być przed typem zwracanym */
_Noreturn void blad_krytyczny(const char *komunikat, int kod) {
    fprintf(stderr, "BLAD KRYTYCZNY: %s\n", komunikat);
    fprintf(stderr, "Kod bledu: %d\n", kod);
    exit(kod);   /* exit() nigdy nie wraca, więc funkcja też nie wraca */
    /* Kompilator wie, że exit() to noreturn, więc nie ostrzega */
}

/* Inny przykład funkcji noreturn */
_Noreturn void zakoncz_sukces(void) {
    puts("Wszystko OK, konczenie...");
    exit(EXIT_SUCCESS);
}

int main(void) {

    /* Rejestracja funkcji sprzątających – wywołane w ODWROTNEJ kolejności */
    /* czyli: komunikat_konca -> zapisz_logi -> sprzataj_baze_danych */
    atexit(sprzataj_baze_danych);   /* zarejestrowana pierwsza, wywołana ostatnia */
    atexit(zapisz_logi);
    atexit(komunikat_konca);        /* zarejestrowana ostatnia, wywołana pierwsza */

    printf("Program dziala...\n");

    /* Symulacja normalnej pracy */
    int sukces = 1;  /* zmień na 0, żeby zobaczyć exit z błędem */

    if (!sukces) {
        /* exit() wywołuje atexit() i kończy program */
        fprintf(stderr, "Cos poszlo nie tak!\n");
        exit(EXIT_FAILURE);   /* wywoła zarejestrowane funkcje i zakończy */
    }

    printf("Operacja zakonczona pomyslnie.\n");

    /* exit(0) jest równoważne return 0 w main */
    /* Oba wywołują atexit() i kończą program */
    /* return EXIT_SUCCESS; */   /* alternatywa */
    exit(EXIT_SUCCESS);          /* oba są równoważne w main */

    /* Ten kod jest nieosiągalny – kompilator może ostrzec */
    printf("To sie nigdy nie wykona.\n");
}
```

```
Zapisywanie logow...

```

(Funkcje `atexit` wywołane w **odwrotnej** kolejności rejestracji.)

```
Zamykanie polaczenia z baza danych...
```

### Typowe błędy i jak ich unikać

**Błąd 1: Używanie liczb zamiast EXIT_SUCCESS / EXIT_FAILURE**
```c
exit(0);    /* działa, ale może być nieprzenośne */
exit(EXIT_SUCCESS);  /* DOBRZE – przenośne i czytelne */
```

**Błąd 2: Zakładanie że po exit() nic się nie dzieje**

```c
/* atexit ZOSTANIE wywołany – jeśli cleanup jest zepsute, exit też będzie zepsuty */
/* Debugowanie: pamiętaj, że atexit może się wykonywać */
```

**Błąd 3: Wywołanie abort() zamiast exit() przez przypadek**

## 11. Podkreślenia w nazwach identyfikatorów

### Teoria

Standard C rezerwuje pewne nazwy identyfikatorów dla kompilatora i biblioteki standardowej. Naruszenie tej rezerwacji może prowadzić do konfliktów z wewnętrznymi mechanizmami.

**Jedno podkreślenie + mała litera: `_nazwa`**
- Zarezerwowane w *globalnej przestrzeni nazw* (pliki nagłówkowe stdlib)
- *Można* używać lokalnie (wewnątrz funkcji), ale jest to odradzane
- Ryzyko: przyszłe wersje standardu mogą wprowadzić globalną `_nazwa`

**Dwa podkreślenia na początku: `__nazwa`**
- Całkowicie zarezerwowane dla kompilatora i implementacji
- **Nigdy** nie należy używać takich nazw we własnym kodzie
- Przykłady: `__FILE__`, `__LINE__`, `__func__`, `__attribute__`

**Jedno podkreślenie + WIELKA litera: `_Nazwa`**
- Zarezerwowane jak `__nazwa`
- Nowe słowa kluczowe C11/C23 często mają tę formę: `_Bool`, `_Complex`, `_Noreturn`, `_Static_assert`
- Nowsze standardy (C23) wprowadzają aliasy bez podkreślenia: `bool`, `noreturn`, `static_assert`

### Kompletny działający kod

```c
#include <stdio.h>     /* printf */
#include <stdlib.h>    /* EXIT_SUCCESS */
#include <stdbool.h>   /* bool (alias dla _Bool, od C99) */
#include <stdnoreturn.h> /* noreturn (alias dla _Noreturn) */

/* Predefiniowane makra kompilatora (używają __, bo są zarezerwowane) */
/* __FILE__ – nazwa pliku źródłowego */
/* __LINE__ – numer bieżącej linii */
/* __func__ – nazwa bieżącej funkcji */
/* __DATE__ – data kompilacji */
/* __TIME__ – czas kompilacji */

/* Konwencja: własne makra piszemy WIELKĄ_LITERĄ, bez podkreśleń na początku */
#define WERSJA_PROGRAMU "1.0.0"
#define MAX_BUFFOR 256

/* Konwencja: własne typy – np. t_prefiks lub NazwaTypu */
typedef struct {
    int x;
    int y;
} Punkt;  /* bez podkreśleń na początku */

/* Zła praktyka – _nazwa w globalnej przestrzeni: */
/* int _zla_globalna = 0; */ /* nie rób tego */

void demonstracja_predefiniowanych(void) {
    /* __func__ – dostępne wewnątrz funkcji */
    printf("Jestem w funkcji: %s\n", __func__);
    printf("Plik: %s\n", __FILE__);
    printf("Linia: %d\n", __LINE__);  /* to wypisze aktualny numer linii */
    printf("Kompilacja: %s %s\n", __DATE__, __TIME__);
}

/* Przykład użycia _Bool / bool */
void demonstracja_bool(void) {
    _Bool flaga1 = 1;    /* stary sposób – _Bool */
    bool  flaga2 = true; /* nowy sposób – bool (wymaga stdbool.h w C99-C17) */
                         /* w C23 bool jest wbudowane bez potrzeby nagłówka */

    printf("flaga1 = %d\n", flaga1);   /* wypisze 1 */
    printf("flaga2 = %d\n", flaga2);   /* wypisze 1 */

    /* _Bool ma specjalną właściwość: zawsze 0 lub 1 */
    _Bool x = 42;   /* x = 1 (nie 42!) */
    printf("_Bool x = 42 -> x = %d\n", x);
}

/* Dobre nazwy identyfikatorów */
int     licznik_iteracji = 0;    /* zrozumiała nazwa, bez podkreśleń na początku */
char    bufor_komunikatu[256];   /* jasna nazwa */
double  temp_celsius;            /* skrót 'temp' jest OK w kontekście */

/* Zły przykład – nazwy kolizyjne z kompilatorami: */
/* int __my_var = 0;  */  /* NIGDY – zarezerwowane */
/* int _MyVar = 0;    */  /* NIGDY – zarezerwowane (_Wielka litera) */

int main(void) {

    printf("Wersja: %s\n", WERSJA_PROGRAMU);

    /* Użycie predefiniowanych makr */
    demonstracja_predefiniowanych();

    /* Użycie bool */
    demonstracja_bool();

    /* Dobre konwencje nazewnicze */
    Punkt p = {3, 4};
    printf("\nPunkt: (%d, %d)\n", p.x, p.y);

    /* _Static_assert – sprawdzenie w czasie kompilacji (C11) */
    /* static_assert to alias (C23) lub z <assert.h> */
    _Static_assert(sizeof(int) >= 2, "int musi miec co najmniej 2 bajty");
    printf("static_assert przeszedl OK\n");

    return EXIT_SUCCESS;
}
```

### Przykładowe wyjście

```
Wersja: 1.0.0
Jestem w funkcji: demonstracja_predefiniowanych
Plik: podkreslenia.c
Linia: 36
Kompilacja: Jun 12 2026 10:30:00
flaga1 = 1
flaga2 = 1
_Bool x = 42 -> x = 1

Punkt: (3, 4)
static_assert przeszedl OK
```

### Typowe błędy i jak ich unikać

**Błąd 1: Własne zmienne globalne z `_`**
```c
int _moj_licznik = 0;  /* ŹLE – może kolidować z biblioteką */
int moj_licznik = 0;   /* DOBRZE */
```

**Błąd 2: Redefiniowanie makr kompilatora**
```c
#define __VERSION__ "moja"  /* NIGDY – zachowanie niezdefiniowane */
```

## 12. Tablice wielowymiarowe – alokacja dynamiczna

### Teoria

Tablica wielowymiarowa to tablica tablic. W C istnieje kilka sposobów jej realizacji, różniących się układem w pamięci i wygodą użycia:

**Sposób 1: Tablica statyczna 2D**
```c
int tab[3][4];  /* 3 wiersze, 4 kolumny – ciągły blok pamięci */
```
Dostęp: `tab[i][j]`. Rozmiar musi być znany w czasie kompilacji.

**Sposób 2: Wskaźnik na tablicę (znany drugi wymiar)**
```c
int (*tab)[4] = malloc(3 * sizeof(int[4]));
```
Jeden ciągły blok pamięci, dostęp: `tab[i][j]`. Drugi wymiar musi być znany.

**Sposób 3: Tablica wskaźników do tablic 1D (jagged array)**
```c
int **tab = malloc(3 * sizeof(int *));
for (int i = 0; i < 3; i++) tab[i] = malloc(4 * sizeof(int));
```
Wiersze mogą mieć różną długość. Dwa osobne bloki pamięci – mniej efektywne pamięciowo (fragmentacja), ale elastyczne.

**Sposób 4: Zlinearyzowana tablica 1D**
```c
int *tab = malloc(3 * 4 * sizeof(int));
/* dostęp: tab[i * 4 + j] */
```
Jeden ciągły blok, eleganckie użycie `calloc`/`malloc`, wymaga ręcznego obliczania indeksów.

**Kiedy co używać:**
- Znany rozmiar w kompilacji → statyczna lub VLA (od C99)
- Elastyczne wiersze różnej długości → tablica wskaźników (sposób 3)
- Wydajność i ciągłość pamięci → zlinearyzowana (sposób 4)
- Prostota przy znanym jednym wymiarze → sposób 2

### Kompletny działający kod

```c
#include <stdio.h>    /* printf */
#include <stdlib.h>   /* malloc, calloc, free, EXIT_SUCCESS */

/* Pomocnicza funkcja wypisująca macierz n x m */
void wypisz_macierz_1d(const int *tab, int n, int m) {
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            printf("%4d", tab[i * m + j]);  /* zlinearyzowany indeks */
        }
        printf("\n");
    }
}

int main(void) {
    int n = 3;   /* liczba wierszy */
    int m = 4;   /* liczba kolumn */

    /* ====================================================
       SPOSÓB 1: Statyczna tablica 2D
       ==================================================== */
    printf("=== Sposob 1: Statyczna tablica 2D ===\n");

    int statyczna[3][4];   /* 3 wiersze, 4 kolumny – rozmiar znany w kompilacji */

    /* Wypełnienie wartościami i*j */
    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 4; j++) {
            statyczna[i][j] = i * 4 + j;  /* row-major: wiersz i, kolumna j */
        }
    }

    /* Wypisanie */
    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 4; j++) {
            printf("%4d", statyczna[i][j]);
        }
        printf("\n");
    }

    /* ====================================================
       SPOSÓB 2: Wskaźnik na tablicę (znany drugi wymiar)
       ==================================================== */
    printf("\n=== Sposob 2: Wskaznik na tablice (znany drugi wymiar) ===\n");

    /* Typ: int (*)[4] – wskaźnik na tablicę 4 intów */
    /* malloc(n * sizeof(int[4])) – alokujemy n wierszy po 4 inty */
    int (*dyn2d)[4] = (int (*)[4])malloc(n * sizeof(int[4]));

    if (dyn2d == NULL) {
        fprintf(stderr, "Blad alokacji!\n");
        return EXIT_FAILURE;
    }

    /* Dostęp identyczny jak do tablicy statycznej */
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < 4; j++) {
            dyn2d[i][j] = i * 10 + j;    /* wypełniamy wartościami */
        }
    }

    /* Alternatywny dostęp przez arytmetykę wskaźnikową */
    **(dyn2d + 1) = 99;    /* dyn2d[1][0] = 99 */

    for (int i = 0; i < n; i++) {
        for (int j = 0; j < 4; j++) {
            printf("%4d", dyn2d[i][j]);
        }
        printf("\n");
    }

    free(dyn2d);   /* jeden free – jeden malloc */

    /* ====================================================
       SPOSÓB 3: Tablica wskaźników do tablic 1D
       ==================================================== */
    printf("\n=== Sposob 3: Tablica wskaznikow (jagged array) ===\n");

    /* Alokujemy tablicę n wskaźników */
    int **jagged = (int **)malloc(n * sizeof(int *));
    if (jagged == NULL) { return EXIT_FAILURE; }

    /* Dla każdego wiersza alokujemy osobny blok */
    for (int i = 0; i < n; i++) {
        jagged[i] = (int *)malloc(m * sizeof(int));  /* m elementów w wierszu i */
        if (jagged[i] == NULL) { return EXIT_FAILURE; }

        /* Wypełniamy wiersz */
        for (int j = 0; j < m; j++) {
            jagged[i][j] = (i + 1) * (j + 1);   /* tabliczka mnożenia */
        }
    }

    /* Dostęp: jagged[wiersz][kolumna] */
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            printf("%4d", jagged[i][j]);
        }
        printf("\n");
    }

    /* Alternatywny dostęp: **(jagged + i) == jagged[i][0] */
    printf("jagged[1][0] = %d, **(jagged+1) = %d\n",
           jagged[1][0], **(jagged + 1));

    /* Zwalnianie – ODWROTNA kolejność: najpierw wiersze, potem tablica wskaźników */
    for (int i = 0; i < n; i++) {
        free(jagged[i]);   /* zwalniamy każdy wiersz */
    }
    free(jagged);          /* zwalniamy tablicę wskaźników */

    /* ====================================================
       SPOSÓB 4: Zlinearyzowana tablica 1D (row-major)
       ==================================================== */
    printf("\n=== Sposob 4: Zlinearyzowana tablica 1D ===\n");

    /* Jeden ciągły blok n*m intów, wyzerowany (calloc) */
    int *linear = (int *)calloc(n * m, sizeof(int));
    if (linear == NULL) { return EXIT_FAILURE; }

    /* Wypełnianie: indeks 2D (i,j) -> indeks 1D (i*m + j) */
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            linear[i * m + j] = i * 100 + j;   /* zlinearyzowany dostęp */
        }
    }

    /* Wypisywanie przez funkcję pomocniczą */
    wypisz_macierz_1d(linear, n, m);

    /* Można też iterować jak po tablicy 1D – to ta sama pamięć */
    printf("Wszystkie elementy linowo: ");
    for (int k = 0; k < n * m; k++) {
        printf("%d ", linear[k]);
    }
    printf("\n");

    /* Makro dla czytelniejszego dostępu do zlinearyzowanej tablicy */
    /* Uwaga: makro używa zmiennej 'm' z zakresu – lepiej użyć funkcji inline */
    #define IDX(i, j) ((i) * m + (j))

    printf("linear[2][1] przez makro IDX: %d\n", linear[IDX(2, 1)]);

    free(linear);   /* jeden free – jeden calloc */

    /* ====================================================
       TABLICE 3D – Sposób jako tablica wskaźników
       ==================================================== */
    printf("\n=== Tablica 3D (wskazniki) ===\n");

    int p = 2;  /* głębokość */

    /* Alokacja p warstw, n wierszy, m kolumn */
    int ***cube = (int ***)malloc(p * sizeof(int **));

    for (int k = 0; k < p; k++) {
        cube[k] = (int **)malloc(n * sizeof(int *));
        for (int i = 0; i < n; i++) {
            cube[k][i] = (int *)malloc(m * sizeof(int));
            for (int j = 0; j < m; j++) {
                cube[k][i][j] = k * 100 + i * 10 + j;
            }
        }
    }

    /* Dostęp: cube[warstwa][wiersz][kolumna] */
    printf("cube[0][1][2] = %d\n", cube[0][1][2]);
    printf("cube[1][2][3] = %d\n", cube[1][2][3]);

    /* Zwalnianie 3D – trzy poziomy free */
    for (int k = 0; k < p; k++) {
        for (int i = 0; i < n; i++) {
            free(cube[k][i]);   /* zwalniamy wiersze */
        }
        free(cube[k]);          /* zwalniamy tablice wskaźników na wiersze */
    }
    free(cube);                 /* zwalniamy tablicę wskaźników na warstwy */

    printf("\nWszystka pamiec zwolniona.\n");

    return EXIT_SUCCESS;
}
```

### Typowe błędy i jak ich unikać

**Błąd 1: Wyciek pamięci przy tablicy wskaźników**
```c
/* Nie można używać jednego free dla jagged array! */
free(jagged);   /* ŹLE – wiersze nadal zajmują pamięć! */

/* DOBRZE: najpierw wiersze, potem struktura */
for (int i = 0; i < n; i++) free(jagged[i]);
free(jagged);
```

**Błąd 2: Zły indeks w zlinearyzowanej tablicy**
```c
int *tab = malloc(n * m * sizeof(int));
tab[i + j];         /* ŹLE – to nie jest [i][j]! */
tab[i * m + j];     /* DOBRZE – row-major: wiersz * liczba_kolumn + kolumna */
```

**Błąd 3: Brak sprawdzania NULL po malloc**
```c
int *tab = malloc(duzo * sizeof(int));
tab[0] = 1;   /* crash jeśli malloc zwrócił NULL (brak pamięci) */

/* DOBRZE */
if (tab == NULL) { /* obsługa błędu */ exit(EXIT_FAILURE); }
```

**Błąd 4: calloc vs malloc – zapomniane zerowanie**
```c
int *tab = malloc(n * sizeof(int));
/* tab może zawierać śmieci! */

int *tab = calloc(n, sizeof(int));
/* tab zawiera same zera – bezpieczniejsze dla macierzy */
```

**Błąd 5: Mieszanie sposobów alokacji z zwalnianiem**
```c
/* Sposób 2 (wskaźnik na tablicę) wymaga jednego free */
/* Sposób 3 (tablica wskaźników) wymaga n+1 free */
/* Nie możesz użyć sposobu 2's free na sposobie 3! */
```
